# خطة تطوير تقرير ما بعد التمرين (Post-Training Report)

> **الهدف:** تقرير احترافي يُلهم المتدرب المبتدئ ويُظهر له قيمة كل مقياس بوضوح

---

## 1. فلسفة التصميم (Design Philosophy)

### المبادئ الأساسية

```
┌─────────────────────────────────────────────────────────────────────┐
│                    User-First Design Principles                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   1️⃣ الصورة أولاً - المتدرب يريد رؤية نفسه                         │
│   ══════════════════════════════════════                            │
│   Hero Section بصورة المتدرب = تحفيز فوري                          │
│   المقاييس تُدمج معها بشكل أنيق (Glass Overlay)                    │
│                                                                      │
│   2️⃣ المعنى قبل الرقم                                               │
│   ═══════════════════════                                           │
│   كل مقياس يظهر مع:                                                 │
│   • القيمة (Value)                                                  │
│   • التصنيف (Good / Needs Work)                                     │
│   • نصيحة قصيرة (One-liner)                                        │
│                                                                      │
│   3️⃣ التدرج في العمق                                                │
│   ════════════════════                                              │
│   Level 1: النظرة السريعة (Quick Glance) ← 3 ثواني                 │
│   Level 2: الفهم الأعمق (Deep Dive) ← للفضوليين                    │
│   Level 3: التحليل الكامل (Full Analysis) ← للجادين                │
│                                                                      │
│   4️⃣ الألوان تحكي القصة                                            │
│   ══════════════════════                                            │
│   🟢 أخضر = ممتاز (90%+)                                            │
│   🟡 أصفر = جيد (70-89%)                                            │
│   🔴 أحمر = يحتاج انتباه (<70%)                                     │
│                                                                      │
│   5️⃣ رسالة واحدة واضحة                                              │
│   ══════════════════════                                            │
│   في أعلى التقرير: "ركز على..." أو "أداء رائع!"                    │
│   هذا أقوى من 10 مقاييس                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. هيكل التقرير (Report Structure)

### التقسيم الذكي حسب نوع الجلسة

```kotlin
// نوع الجلسة يحدد شكل التقرير
enum class SessionType {
    SINGLE_EXERCISE,    // تمرين واحد (Squats only)
    WORKOUT             // مجموعة تمارين (Leg Day)
}

// التقرير الموحد (تحديث PostTrainingReport الموجود)
data class PostTrainingReport(
    // ... الحقول الموجودة ...
    
    // إضافات جديدة
    val sessionType: SessionType,
    val heroImage: String?,                    // صورة المتدرب الرئيسية
    val quickInsight: QuickInsight,            // الرسالة الرئيسية
    val performanceMetrics: PerformanceMetrics, // المقاييس المحسوبة
    val workoutDetails: WorkoutDetails?        // تفاصيل الـ Workout (إن وجد)
)

// الرسالة الرئيسية في أعلى التقرير
data class QuickInsight(
    val type: InsightType,           // CELEBRATION | FOCUS_POINT | DANGER_WARNING
    val title: LocalizedText,        // "أداء رائع!" أو "ركز على..."
    val subtitle: LocalizedText,     // تفصيل بسيط
    val actionable: LocalizedText?   // نصيحة قابلة للتنفيذ
)

enum class InsightType {
    CELEBRATION,      // أداء ممتاز - احتفال
    FOCUS_POINT,      // أداء جيد - نقطة تركيز
    DANGER_WARNING    // تحذير - يحتاج انتباه
}
```

object MetricsDisplayRules {
    
    // متى نعرض Form Consistency؟
    fun shouldShowFormConsistency(repsCount: Int): Boolean {
        return repsCount >= 4  // أقل من 4 عدات = لا يوجد ما يكفي للمقارنة
    }
    
    // متى نعرض Fatigue Index؟
    fun shouldShowFatigueIndex(repsCount: Int, fatigueIndex: Int?): Boolean {
        return repsCount >= 6 && fatigueIndex != null
    }
    
    // متى نعرض Symmetry؟
    fun shouldShowSymmetry(exerciseType: ExerciseType): Boolean {
        return exerciseType.isBilateral  // فقط للتمارين الثنائية
    }
    
    // متى نعرض Load Metrics؟
    fun shouldShowLoadMetrics(weightKg: Float?): Boolean {
        return weightKg != null && weightKg > 0
    }
    
    // ما هي الرسالة الرئيسية (Quick Insight)؟
    fun getQuickInsight(report: PostTrainingReport): QuickInsight {
        return when {
            report.dangerAlerts.isNotEmpty() -> 
                QuickInsight(DANGER_WARNING, "انتبه", dangerMessage)
            report.summary.averageScore >= 90 -> 
                QuickInsight(CELEBRATION, "أداء رائع!", celebrationMessage)
            else -> 
                QuickInsight(FOCUS_POINT, "ركز على", focusMessage)
        }
    }
}
