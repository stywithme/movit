# خطة تحليل التمرينات من الفيديوهات

## نظرة عامة

اضافة ميزة تحليل التمرينات من خلال فيديوهات يرفعها المستخدم، مع الحفاظ على نفس تجربة التحليل الحية (الكاميرا) واضافة امكانيات التحكم في الفيديو.

### المتطلبات الاساسية:
- تحليل الفيديو اثناء تشغيله (Real-time playback)
- رفع فيديو من الجاليري
- ايقاف/تشغيل + Seek bar
- حفظ النتائج (العدات، الجودة، الاخطاء، الارشادات)
- نفس شاشة التدريب (TrainingActivity) مع وضعين: كاميرا/فيديو
- نفس منطق TrainingEngine المستخدم للكاميرا

### 1. VideoManager.kt

```kotlin
/**
 * VideoManager - ادارة تشغيل الفيديو واستخراج الاطارات
 * 
 * يستخدم ExoPlayer للتشغيل + TextureView لاستخراج الاطارات
 * 
 * المسؤوليات:
 * - تحميل الفيديو من URI
 * - التحكم في التشغيل (play/pause/seek)
 * - استخراج الاطارات عبر TextureView.getBitmap()
 * - ارسال الاطارات للتحليل
 */
class VideoManager(
    private val context: Context,
    private val textureView: TextureView,
    private val onFrameAvailable: (Bitmap, Long) -> Unit,
    private val onPlaybackStateChanged: (PlaybackState) -> Unit,
    private val onProgressChanged: (currentMs: Long, durationMs: Long) -> Unit,
    private val onSeekPerformed: () -> Unit  // لاعلام Activity بحدوث seek
) {
    
    enum class PlaybackState {
        IDLE,
        LOADING,
        READY,
        PLAYING,
        PAUSED,
        ENDED,
        ERROR
    }
    
    private var player: ExoPlayer? = null
    private var frameExtractorJob: Job? = null
    
    /**
     * تحميل الفيديو
     */
    fun loadVideo(uri: Uri) {
        player = ExoPlayer.Builder(context).build().apply {
            setVideoTextureView(textureView)
            setMediaItem(MediaItem.fromUri(uri))
            prepare()
            
            addListener(object : Player.Listener {
                override fun onPlaybackStateChanged(state: Int) {
                    val playbackState = when (state) {
                        Player.STATE_IDLE -> PlaybackState.IDLE
                        Player.STATE_BUFFERING -> PlaybackState.LOADING
                        Player.STATE_READY -> PlaybackState.READY
                        Player.STATE_ENDED -> PlaybackState.ENDED
                        else -> PlaybackState.ERROR
                    }
                    onPlaybackStateChanged(playbackState)
                }
            })
        }
    }
    
    /**
     * بدء التشغيل مع استخراج الاطارات
     */
    fun play() {
        player?.play()
        startFrameExtraction()
    }
    
    fun pause() {
        player?.pause()
        stopFrameExtraction()
    }
    
    /**
     * Seek مع اعلام Activity لاعادة ضبط الحالة
     */
    fun seekTo(positionMs: Long) {
        player?.seekTo(positionMs)
        onSeekPerformed()  // Activity ستعيد ضبط PoseLandmarker و TrainingEngine
    }
    
    /**
     * استخراج الاطارات من TextureView
     * يتم استدعاؤها بمعدل الفيديو الاصلي
     */
    private fun startFrameExtraction() {
        frameExtractorJob = CoroutineScope(Dispatchers.Default).launch {
            while (isActive && player?.isPlaying == true) {
                val bitmap = textureView.getBitmap()
                val timestampMs = player?.currentPosition ?: 0L
                
                if (bitmap != null) {
                    withContext(Dispatchers.Main) {
                        onFrameAvailable(bitmap, timestampMs)
                        onProgressChanged(timestampMs, player?.duration ?: 0L)
                    }
                }
                
                // انتظار حتى الاطار التالي (~33ms لـ 30fps)
                delay(33)
            }
        }
    }
    
    private fun stopFrameExtraction() {
        frameExtractorJob?.cancel()
    }
    
    fun getDuration(): Long = player?.duration ?: 0L
    fun getCurrentPosition(): Long = player?.currentPosition ?: 0L
    fun isPlaying(): Boolean = player?.isPlaying ?: false
    
    fun release() {
        stopFrameExtraction()
        player?.release()
        player = null
    }
}
```

```kotlin
/**
 * AnalysisResultStorage - حفظ واسترجاع نتائج التحليل
 * يستخدم Gson + Internal Storage (بدون مكتبات جديدة)
 */
class AnalysisResultStorage(private val context: Context) {
    
    private val gson = Gson()
    private val storageDir = File(context.filesDir, "analysis_results")
    
    init {
        if (!storageDir.exists()) {
            storageDir.mkdirs()
        }
    }
    
    /**
     * حفظ نتيجة التحليل
     */
    fun save(result: VideoAnalysisResult): Boolean {
        return try {
            val file = File(storageDir, "${result.id}.json")
            file.writeText(gson.toJson(result))
            true
        } catch (e: Exception) {
            Log.e("AnalysisResultStorage", "Failed to save: ${e.message}")
            false
        }
    }
    
    /**
     * استرجاع نتيجة بالـ ID
     */
    fun getById(id: String): VideoAnalysisResult? {
        return try {
            val file = File(storageDir, "$id.json")
            if (file.exists()) {
                gson.fromJson(file.readText(), VideoAnalysisResult::class.java)
            } else null
        } catch (e: Exception) {
            null
        }
    }
    
    /**
     * استرجاع كل النتائج
     */
    fun getAll(): List<VideoAnalysisResult> {
        return storageDir.listFiles()
            ?.filter { it.extension == "json" }
            ?.mapNotNull { file ->
                try {
                    gson.fromJson(file.readText(), VideoAnalysisResult::class.java)
                } catch (e: Exception) { null }
            }
            ?.sortedByDescending { it.analysisDate }
            ?: emptyList()
    }
    
    /**
     * حذف نتيجة
     */
    fun delete(id: String): Boolean {
        return File(storageDir, "$id.json").delete()
    }
}
```

### 4. تعديل PoseLandmarkerHelper.kt

```kotlin
// تصحيح: استخدام detectForVideo() بدلا من detect()

class PoseLandmarkerHelper(...) {
    
    enum class RunMode {
        LIVE_STREAM,  // للكاميرا (الوضع الحالي)
        VIDEO         // للفيديو (الوضع الجديد)
    }
    
    private var currentRunMode = RunMode.LIVE_STREAM
    private var videoModeLandmarker: PoseLandmarker? = null
    
    /**
     * تهيئة للفيديو (VIDEO mode)
     */
    fun initializeForVideo(modelType: ModelType = ModelType.FULL, useGpu: Boolean = true) {
        currentRunMode = RunMode.VIDEO
        
        val baseOptionsBuilder = BaseOptions.builder()
            .setModelAssetPath(modelType.fileName)
            
        if (useGpu) {
            baseOptionsBuilder.setDelegate(Delegate.GPU)
        }
        
        val options = PoseLandmarker.PoseLandmarkerOptions.builder()
            .setBaseOptions(baseOptionsBuilder.build())
            .setRunningMode(RunningMode.VIDEO)  // VIDEO mode (لا LIVE_STREAM)
            .setNumPoses(NUM_POSES)
            .setMinPoseDetectionConfidence(MIN_POSE_DETECTION_CONFIDENCE)
            .setMinTrackingConfidence(MIN_POSE_TRACKING_CONFIDENCE)
            .setMinPosePresenceConfidence(MIN_POSE_PRESENCE_CONFIDENCE)
            // لا نستخدم setResultListener في VIDEO mode
            .build()
            
        videoModeLandmarker = PoseLandmarker.createFromOptions(context, options)
    }
    
    /**
     * تحليل اطار فيديو (متزامن - synchronous)
     * 
     * مهم: يستخدم detectForVideo() وليس detect()
     * - detect() -> لـ IMAGE mode (صورة واحدة)
     * - detectForVideo() -> لـ VIDEO mode (اطارات متتابعة)
     * - detectAsync() -> لـ LIVE_STREAM mode (الكاميرا)
     */
    fun detectPoseFromBitmap(bitmap: Bitmap, timestampMs: Long): PoseResult? {
        if (currentRunMode != RunMode.VIDEO) {
            throw IllegalStateException("Use detectPose() for LIVE_STREAM mode")
        }
        
        val mpImage = BitmapImageBuilder(bitmap).build()
        
        // detectForVideo() - الدالة الصحيحة لـ VIDEO mode
        val result = videoModeLandmarker?.detectForVideo(mpImage, timestampMs)
        
        return result?.let { convertToResult(it, bitmap.width, bitmap.height) }
    }
    
    /**
     * اعادة تهيئة الـ Landmarker (عند الـ Seek)
     * 
     * مهم جدا: MediaPipe VIDEO mode يتوقع timestamps متزايدة
     * عند الـ Seek للخلف، لازم اعادة انشاء الـ landmarker
     */
    fun resetForVideo() {
        videoModeLandmarker?.close()
        initializeForVideo()
    }
    
    private fun convertToResult(
        result: PoseLandmarkerResult, 
        width: Int, 
        height: Int
    ): PoseResult? {
        if (result.landmarks().isEmpty()) return null
        
        val landmarks = result.landmarks()[0]
        val worldLandmarks = if (result.worldLandmarks().isNotEmpty()) {
            result.worldLandmarks()[0]
        } else null
        
        return PoseResult(
            landmarks = landmarks,
            worldLandmarks = worldLandmarks,
            timestampMs = System.currentTimeMillis(),
            inferenceTimeMs = 0,
            imageWidth = width,
            imageHeight = height,
            modelType = currentModelType.displayName
        )
    }
    
    fun closeVideoMode() {
        videoModeLandmarker?.close()
        videoModeLandmarker = null
    }
}
```

```kotlin
class TrainingActivity : AppCompatActivity(), PoseLandmarkerHelper.PoseDetectionListener {
    
    companion object {
        const val EXTRA_TRAINING_MODE = "training_mode"
        const val EXTRA_VIDEO_URI = "video_uri"
        
        const val MODE_CAMERA = "camera"
        const val MODE_VIDEO = "video"
    }
    
    private var trainingMode: String = MODE_CAMERA
    private var videoUri: Uri? = null
    private var videoManager: VideoManager? = null
    
    // Timestamp tracker لاكتشاف الـ Seek
    private var lastVideoTimestamp: Long = 0L
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        trainingMode = intent.getStringExtra(EXTRA_TRAINING_MODE) ?: MODE_CAMERA
        videoUri = intent.getParcelableExtra(EXTRA_VIDEO_URI)
        
        // ...existing setup...
        
        when (trainingMode) {
            MODE_CAMERA -> {
                setupCameraMode()
            }
            MODE_VIDEO -> {
                setupVideoMode()
            }
        }
    }
    
    private fun setupVideoMode() {
        // اخفاء Camera Preview واظهار Video TextureView
        binding.previewView.visibility = View.GONE
        binding.videoTextureView.visibility = View.VISIBLE
        binding.videoControls.visibility = View.VISIBLE
        
        // اخفاء setupPosePanel للفيديو
        binding.setupPosePanel.visibility = View.GONE
        
        // تهيئة PoseLandmarker لوضع VIDEO
        poseLandmarkerHelper?.initializeForVideo()
        
        // تهيئة VideoManager
        videoManager = VideoManager(
            context = this,
            textureView = binding.videoTextureView,
            onFrameAvailable = { bitmap, timestampMs ->
                processVideoFrame(bitmap, timestampMs)
            },
            onPlaybackStateChanged = { state ->
                handlePlaybackStateChanged(state)
            },
            onProgressChanged = { current, duration ->
                updateSeekBar(current, duration)
            },
            onSeekPerformed = {
                handleSeekPerformed()
            }
        )
        
        videoUri?.let { videoManager?.loadVideo(it) }
    }
    
    /**
     * معالجة اطار الفيديو
     */
    private fun processVideoFrame(bitmap: Bitmap, timestampMs: Long) {
        val poseResult = poseLandmarkerHelper?.detectPoseFromBitmap(bitmap, timestampMs)
        
        if (poseResult != null) {
            // نفس منطق الكاميرا بالضبط
            val smoothedLandmarks = landmarkSmoother.smooth(
                poseResult.landmarks,
                timestampMs
            )
            
            val angles = AngleCalculator.calculateAllAnglesSmoothed(
                smoothedLandmarks,
                visibilityThreshold = 0.3f
            )
            
            trainingEngine?.processFrame(angles, smoothedLandmarks)
            
            // Update UI
            binding.skeletonOverlay.updateWithArrowInfos(
                smoothedLandmarks = smoothedLandmarks,
                inputImageWidth = bitmap.width,
                inputImageHeight = bitmap.height,
                angles = angles,
                arrowInfos = trainingEngine?.arrowInfos?.value ?: emptyMap(),
                positionErrors = trainingEngine?.positionErrors?.value ?: emptyList()
            )
        }
        
        lastVideoTimestamp = timestampMs
    }
    
    /**
     * معالجة الـ Seek - اعادة ضبط الحالة
     */
    private fun handleSeekPerformed() {
        // 1. اعادة ضبط PoseLandmarker (مطلوب لـ VIDEO mode)
        poseLandmarkerHelper?.resetForVideo()
        
        // 2. اعادة ضبط LandmarkSmoother
        landmarkSmoother.reset()
        
        // 3. اعادة ضبط TrainingEngine
        // قرار: عند الـ Seek، نعيد ضبط التحليل من البداية
        trainingEngine?.stop()
        trainingEngine?.start()
        
        // 4. اعادة ضبط timestamp tracker
        lastVideoTimestamp = 0L
    }
}
```

## التفاصيل التقنية

### 1. اختيار مكتبة تشغيل الفيديو

**الخيار الموصى به: ExoPlayer (Media3)**

```kotlin
// build.gradle.kts
dependencies {
    implementation("androidx.media3:media3-exoplayer:1.5.0")
    implementation("androidx.media3:media3-ui:1.5.0")
    implementation("androidx.media3:media3-common:1.5.0")
}
```

**الاسباب:**
- مدعومة من Google ومتكاملة مع Android
- تدعم معظم تنسيقات الفيديو
- اداء ممتاز

### 2. استخراج الاطارات من الفيديو

**الطريقة المستخدمة: TextureView + getBitmap()**

```kotlin
// TextureView يسمح باستخراج الاطار الحالي كـ Bitmap
val bitmap = textureView.getBitmap()
```

**لماذا TextureView وليس SurfaceView؟**
- SurfaceView لا يدعم getBitmap() مباشرة
- TextureView يعمل مع OpenGL ويسمح بالتحويل لـ Bitmap
- ExoPlayer يدعم كليهما: setVideoTextureView() / setVideoSurfaceView()

**ملاحظة:** VideoFrameMetadataListener يعطي metadata فقط (timestamp, format) وليس الـ Bitmap نفسه.

### 3. MediaPipe VIDEO mode - التفاصيل المهمة

```kotlin
// الفرق بين الوضعين:

// LIVE_STREAM mode (الكاميرا):
// - detectAsync() - غير متزامن
// - يحتاج ResultListener
// - يتحمل timestamps غير متسقة

// VIDEO mode (الفيديو):
// - detectForVideo() - متزامن
// - يرجع النتيجة مباشرة
// - يتوقع timestamps متزايدة دائما!
```

### 4. سلوك الـ Seek

**مشكلة الـ Seek:**
MediaPipe VIDEO mode يتوقع ان الـ timestamps تكون متزايدة دائما. عند الـ Seek للخلف (او القفز لنقطة سابقة)، الـ timestamps ستكون اقل من السابقة -> نتائج غير متوقعة او اخطاء.

**الحل:**
عند كل Seek، نعيد تهيئة الـ PoseLandmarker:

```kotlin
fun handleSeekPerformed() {
    // 1. اغلاق الـ landmarker الحالي
    poseLandmarkerHelper?.closeVideoMode()
    
    // 2. اعادة انشائه
    poseLandmarkerHelper?.initializeForVideo()
    
    // 3. اعادة ضبط المكونات الاخرى
    landmarkSmoother.reset()
    trainingEngine?.stop()
    trainingEngine?.start()
}
```

### 5. الفرق بين الكاميرا والفيديو

| الجانب | الكاميرا (LIVE_STREAM) | الفيديو (VIDEO) |
|--------|------------------------|-----------------|
| RunningMode | LIVE_STREAM | VIDEO |
| Detection API | detectAsync() | detectForVideo() |
| Sync/Async | Async (callback) | Sync (return) |
| Timestamp | SystemClock.uptimeMillis() | Video presentation time |
| Setup Pose | مطلوب | غير مطلوب |
| Countdown | 3-2-1 | غير مطلوب |
| Controls | Pause only | Play/Pause/Seek |
| Seek Handling | N/A | Reset landmarker + engine |

## تحديات محتملة وحلولها

### 1. الاداء مع الفيديوهات عالية الدقة

**المشكلة:** فيديو 4K قد يبطئ التحليل

**الحل:**
```kotlin
// تصغير الاطار قبل التحليل
fun resizeForAnalysis(bitmap: Bitmap, maxDimension: Int = 640): Bitmap {
    val ratio = maxDimension.toFloat() / maxOf(bitmap.width, bitmap.height)
    if (ratio >= 1f) return bitmap
    
    return Bitmap.createScaledBitmap(
        bitmap,
        (bitmap.width * ratio).toInt(),
        (bitmap.height * ratio).toInt(),
        true
    )
}
```

### 2. Memory Management

**المشكلة:** تحميل كثير من الاطارات يستهلك الذاكرة

**الحل:**
```kotlin
// اعادة استخدام نفس الـ Bitmap buffer
private var bitmapBuffer: Bitmap? = null

fun processFrame(newBitmap: Bitmap) {
    // Recycle القديم
    if (bitmapBuffer != newBitmap) {
        bitmapBuffer?.recycle()
    }
    bitmapBuffer = newBitmap
    
    // ... process ...
}
```

### 3. Orientation Issues

**المشكلة:** بعض الفيديوهات مقلوبة/مدورة

**الحل:**
```kotlin
// قراءة rotation metadata وتطبيقه
fun getVideoRotation(uri: Uri): Int {
    val retriever = MediaMetadataRetriever()
    retriever.setDataSource(context, uri)
    val rotation = retriever.extractMetadata(
        MediaMetadataRetriever.METADATA_KEY_VIDEO_ROTATION
    )?.toIntOrNull() ?: 0
    retriever.release()
    return rotation
}

// تطبيق الـ rotation على الـ Bitmap
fun rotateBitmap(bitmap: Bitmap, degrees: Int): Bitmap {
    if (degrees == 0) return bitmap
    val matrix = Matrix().apply { postRotate(degrees.toFloat()) }
    return Bitmap.createBitmap(bitmap, 0, 0, bitmap.width, bitmap.height, matrix, true)
}
```
