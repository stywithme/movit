# Content Readiness (for first 20 users)

## الهدف

كل المحتوى المطلوب جاهزاً وصحيحاً قبل بدء الـ 20 مستخدماً.

## معمارية المحتوى (حقائق من الكود)

- المحتوى **server-driven**: يُؤلَّف في الـ backend ويُدار عبر الـ Admin Dashboard، والأندرويد يستهلكه عبر `ExerciseRepository` (+ cache + offline fallback).
- التمارين لها schema موثّق: `Docs/00-Active-Reference/Contracts/Exercise-JSON-Schema.md` → `ExerciseConfig`.
- الـ assets لا تحتوي محتوى تمارين جاهز (فقط `desk_test.json` كـ fallback) — كل المحتوى الحقيقي من الباك إند.
- التقييم/القوالب تُدار في `/admin/assessment-templates`؛ البرامج في موديول programs.

## مبدأ الأولوية

المقياس = استمرار أسبوعين + تطور (ملف 16). لذلك:

> **الحد الأدنى الحرج = محتوى أول أسبوعين كامل وصحيح + التقييم + إعادة التقييم.**

الأسبوع 3-4 يُجهَّز بالتوازي، ولا يوقف بدء التجربة.

## مستويات المحتوى

### المستوى 1: مكتبة التمارين (الأساس)

لكل تمرين (`ExerciseConfig`، انظر `Exercise-JSON-Schema.md`):

- **الهوية**: `name`، `description`، `instructions` (ar/en).
- **التصنيف**: `category`، `muscles`، `equipment`، `tags`، `countingMethod` (up_down / push_pull / hold).
- **إعداد الوضعية** (`poseVariants`): `cameraPosition` (side/front/back)، `trackedJoints` مع **عتبات الزوايا** (PERFECT → NORMAL → PAD → WARNING → DANGER)، و `difficultyLevels` (لازم `beginner` وإلا `TrainingEngine` يرمي exception).
- **رسائل التصحيح** (`feedbackMessages`): motivational / common_mistake / tip + رسائل لكل حالة مفصل (ar/en).
- **وسائط العرض**: فيديو/أنميشن/صورة للـ instructions.

> الأصعب والأهم: **عتبات الزوايا لكل مفصل**. هي قلب التصحيح، وتتطلب ضبطاً واختباراً على حركة حقيقية، لا مجرد كتابة. أي تمرين في البرنامج لازم يعدّي `ExerciseLoader.validateStateRanges` بدون errors.

### المستوى 2: برنامج 28 يوم

- البنية: 4 أسابيع × 3 أيام ≈ 12 جلسة.
- كل جلسة (`ProgramSessionItem`): تمارين مرتّبة + sets + reps/duration + rest + إحماء + الدقائق التقديرية (20-30).
- تدرّج بسيط عبر الأسابيع + تعريف أيام الراحة.
- **يستخدم فقط تمارين جاهزة (مستوى 1).**
- عدد تقديري: ~15-25 تمرين عبر الشهر، ~6-10 في أول أسبوعين.

### المستوى 3: التقييم + إعادة التقييم

- **التقييم الأولي**: قالب (`AssessmentTemplate` + تماريـنه) + عتبات المستوى (`domainWeights`، level ranges) → يحدد `UserLevelProfile`.
- **فحص الأهلية (Safety Gate)**: أسئلة `PreScreening` + قواعد red-flags + نصوص الإحالة لمختص.
- **إعادة التقييم (progression)**: قالب المقارنة like-for-like عند الأسبوعين — **حرج لإثبات التطور** (ملف 16).
- **نص النتيجة**: شرح المستوى و"ليه"، والخطوة التالية، بمعنى لا أرقام فقط.

### المستوى 4: Onboarding ولحظة الـ Aha

- أسئلة الـ onboarding + "ليه بنسأل" (الهدف، النشاط، الوقت، الأيام، الطول/الوزن، قيود/ألم).
- نص الخصوصية ("الفيديو لا يخرج من جهازك") قبل أول فتح كاميرا.
- محتوى "جرّب حركة واحدة" (تمرين سهل واحد لـ Aha مصغّر — ملف 19).

### المستوى 5: نصوص التدريب والتصحيح (Coaching Copy)

- رسائل تصحيح فورية لكل مفصل/حالة (`MobileMessageResolver` + `feedbackMessages`).
- جُمل TTS منطوقة (`TtsVoiceSelector`) + cues الإعداد/العد التنازلي/العدّات.
- النبرة: "أعد المحاولة" لا "أنت غلط"، تشجيعية، **عامية مصرية**.
- مرجع: `Docs/03-Implemented-Archive/Feedback/Feedback-Messages-Refactor-Plan.md`، `Docs/03-Implemented-Archive/Feedback/Feedback-System-Improvement-Plan.md`.

### المستوى 6: نصوص التقارير (الأرقام → معنى) — Loop 1

- تقرير الجلسة: جُمل معنى ("ثباتك تحسن"، "أكثر خطأ متكرر: …").
- التقرير الأسبوعي.
- صياغة التطور نسبةً لخط الأساس (ملف 16).
- نصوص `PerformanceRating` (EXCELLENT/GOOD/FAIR/POOR) بالعربي.

### المستوى 7: نصوص الإشعارات والتنبيهات — Loop 3/4

- نص الـ push (تذكير يوم التمرين، streak_at_risk) — **بعد بناء Loop 4** (ملف 18).
- نصوص `HomeAlertData` (reassessment_due / progression_applied / level_up / streak_at_risk) ar/en.
- نصوص الـ streak و"تمرين اليوم".

### المستوى 8: اللغة والنبرة

- كل المحتوى **عامية مصرية** (الكوهورت مصري)، مراجَع للنبرة.
- en كثانوي. `strings.xml` (ar) موجود — يُراجَع للميزات الجديدة.
- الكوهورت فيه سيدات (اية، ايمان، نور، سارة، زينب) → نبرة محايدة جندرياً + طمأنة الخصوصية مهمة بشكل خاص.

### المستوى 9: الثقة والسلامة (حد أدنى للبيتا)

حتى مع المعارف:

- إشعار خصوصية مختصر (الفيديو على الجهاز).
- تنويه Wellness لا تشخيص طبي.
- نصوص الإحالة لمختص عند red-flags.

## معايير الجاهزية (Definition of Ready)

تمرين جاهز فقط إذا:

- يعدّي `validateStateRanges` بدون errors.
- فيه `difficultyLevels` يحتوي `beginner`.
- رسائل التصحيح مكتوبة (ar) لكل حالة مفصل أساسي.
- له وسيط عرض (فيديو/صورة).
- جُرّب فعلياً بالكاميرا مرة على الأقل (الزوايا منطقية).

برنامج جاهز فقط إذا: كل جلساته تُحمّل، وكل تمارينها جاهزة، والدقائق ≈ 20-30، والصعوبة beginner.

التقييم جاهز فقط إذا: initial + reassessment يُحَلّان لملف مبتدئ، والنتيجة فيها نص معنى لا أرقام فقط.

## قائمة فحص قبل الإطلاق (Pre-launch)

- [ ] تمارين أول أسبوعين (6-10) جاهزة بالكامل (ranges + feedback ar + media).
- [ ] برنامج الأسبوعين 1-2 مبني ويُحمّل بالكامل.
- [ ] قالب التقييم الأولي يعمل لملف مبتدئ.
- [ ] قالب إعادة التقييم (الأسبوعين) جاهز — like-for-like.
- [ ] Safety Gate: أسئلة + قواعد + نصوص إحالة.
- [ ] Onboarding: أسئلة + نص خصوصية + تمرين الـ Aha.
- [ ] نصوص تقرير الجلسة والأسبوع (معنى لا أرقام).
- [ ] نص الخصوصية + تنويه Wellness.
- [ ] مراجعة نبرة عامية مصرية + محايدة جندرياً.
- [ ] باقي برنامج الأسبوع 3-4 (يكتمل أثناء التجربة).
- [ ] نصوص الإشعارات (مع بناء Loop 4).
