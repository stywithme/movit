# MyFatoorah — نظرة عامة على منصة الدفع

> المصدر: [MyFatoorah API Documentation](https://docs.myfatoorah.com/docs/get-started)

## المقدمة

MyFatoorah منصة دفع إقليمية تدعم عدة دول (الكويت، السعودية، الإمارات، مصر، قطر، البحرين، الأردن، عُمان). توفر واجهات برمجية متعددة لتناسب احتياجات الأعمال المختلفة.

## طرق التكامل

| الطريقة | الوصف | الاستخدام الموصى به |
|---------|-------|---------------------|
| **Embedded Payment** | الدفع مباشرة داخل صفحة الدفع الخاصة بك | ✅ موصى به |
| **Hosted Payment Page** | إعادة توجيه العميل لصفحة MyFatoorah لإتمام الدفع | مناسب للبساطة |
| **Invoicing / Payment Links** | إنشاء رابط دفع وإرساله عبر البريد أو SMS | مناسب للفواتير والروابط |
| **Direct Payment** | إرسال بيانات البطاقة مباشرة (يتطلب شهادة PCI) | للمؤسسات الكبيرة |

## مقارنة الطرق

| الميزة | Embedded | Hosted Page | Invoicing |
|--------|----------|-------------|-----------|
| التضمين في موقعك | ✅ | ❌ | ❌ |
| إعادة التوجيه | لبعض الطرق فقط | ✅ | ✅ |
| امتثال PCI | تُدار من MyFatoorah | تُدار من MyFatoorah | تُدار من MyFatoorah |
| طرق الدفع | بطاقات ومحافظ + إعادة توجيه | كل الطرق المفعلة | كل الطرق المفعلة |
| حفظ البطاقة | نعم | نعم | نعم |

## تحديث حالة الطلب في نظامك

- **Webhook**: إشعارات فورية من الخادم إلى الخادم (موصى به)
- **Get Payment Details**: استعلام عن حالة الدفع عبر `PaymentId` بعد إعادة التوجيه

> يُوصى باستخدام **الاثنين معاً** لضمان تحديث حالة الطلب بشكل موثوق. راجع [Updating Payment Status Guidelines](https://docs.myfatoorah.com/docs/v3-updating-payment-status-guidelines).
