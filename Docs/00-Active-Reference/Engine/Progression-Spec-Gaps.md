# Progression Spec Gaps — نظام التقدم والتوصيف

> مبني على دراسة ACSM 2026 + نقاشات التصميم + الحالة الفعلية للمشروع.
> المبدأ: **لا اختراع — فقط تطبيق الأساس العلمي بشكل واضح وبسيط.**
> للتنفيذ الفعلي اليوم راجع [trainee-journey](../Architecture-As-Built/trainee-journey-current-state/README.md) و [`progression.service.ts`](../../../backend/src/modules/progression/progression.service.ts).

## الخطوة 1: تقسيم التمارين بشكل صحيح

### التقسيم العلمي المبسط

كل تمرين في النظام يقع في واحد من **4 أنواع** فقط:

| النوع | Archetype Code | كيف يتقدم | أمثلة |
|-------|---------------|-----------|-------|
| **Weighted Rep** | `weighted_rep` | وزن + تكرارات | Squat, Deadlift, Bench Press, Overhead Press, Barbell Row |
| **Bodyweight Rep** | `bodyweight_rep` | تكرارات + صعوبة | Push-up, Pull-up, Air Squat, Lunge, Burpee, Dip |
| **Weighted Hold** | `weighted_hold` | مدة + وزن | Weighted Plank, Farmer's Walk, Overhead Hold |
| **Bodyweight Hold** | `bodyweight_hold` | مدة + صعوبة | Plank, Wall Sit, Dead Hang, L-Sit |

**كيف نحدد النوع:**

```
if (countingMethod === 'HOLD') {
    if (supportsWeight) → weighted_hold
    else               → bodyweight_hold
}
else { // UP_DOWN
    if (supportsWeight) → weighted_rep
    else               → bodyweight_rep
}
```

**هذه الحقول موجودة بالفعل** على `Exercise`: `countingMethod` و `supportsWeight`.

## الخطوة 2: نظام الهدف (Training Goal)

### الأهداف المتاحة

| الهدف | الاسم بالعربي | لمن |
|-------|--------------|-----|
| `STRENGTH` | قوة عضلية | يريد يرفع أثقل |
| `HYPERTROPHY` | تضخم عضلي | يريد عضلات أكبر |
| `POWER` | قدرة انفجارية | يريد سرعة وقوة معاً |
| `GENERAL_HEALTH` | لياقة عامة | يريد صحة أفضل |

### أين يُحفظ الهدف

- حقل `trainingGoal` على `User` (يُختار في Onboarding أو Settings).
- يمكن تجاوزه على `ActivePlan` لو المستخدم غيّر هدفه مؤقتاً.

### كيف يؤثر الهدف

الهدف يغير **3 أشياء فقط**:
1. **ماذا يُقترح** (وزن، تكرارات، مدة).
2. **ترتيب أولوية التقدم** (أي محور يتقدم أولاً).
3. **الرسائل والتنبيهات** (ما يراه المستخدم).

## الخطوة 3: كيف يتقدم كل نوع تمرين حسب كل هدف

### النوع 1: Weighted Rep (الأهم والأكثر شيوعاً)

**المبدأ العلمي:** est1RM يُحسب بعد كل جلسة → الوزن المقترح للجلسة القادمة = est1RM × النسبة حسب الهدف.

| الهدف | %1RM المقترح | نطاق التكرارات | المجموعات | أولوية التقدم |
|-------|-------------|----------------|-----------|---------------|
| **STRENGTH** | 80–85% | 5–8 | 3–4 | load ← reps ← sets |
| **HYPERTROPHY** | 65–75% | 8–12 | 3–5 | sets ← reps ← load |
| **POWER** | 40–60% | 3–6 | 3–4 | load (ضمن النطاق) |
| **GENERAL_HEALTH** | 60–70% | 10–15 | 2–3 | reps ← sets ← load |

**مسار الاقتراح التلقائي:**

```
آخر جلسة: Squat 60kg × 10 reps
→ est1RM = 80kg (معادلة Epley)

الهدف: STRENGTH
→ وزن مقترح = 80 × 0.82 = 65.5 → تقريب إلى 65kg
→ تكرارات مستهدفة: 6–8
→ المجموعات: 3

الهدف: HYPERTROPHY
→ وزن مقترح = 80 × 0.70 = 56kg
→ تكرارات مستهدفة: 10–12
→ المجموعات: 4
```

**المستخدم يقدر يعدّل الوزن يدوياً دائماً.**

**متى يتقدم (Promotion):**
- أكمل جلستين متتاليتين بـ:
  - formScore ≥ 70%
  - إتمام كل التكرارات المستهدفة (completionRate ≥ 85%)
  - ROM ≥ 75%
- → النظام يرفع المحور الأول في الأولوية:
  - **STRENGTH**: يرفع الوزن (+2.5kg) ← est1RM يتحدث تلقائياً
  - **HYPERTROPHY**: يرفع المجموعات (+1) حتى الحد → ثم التكرارات → ثم الوزن

### النوع 2: Bodyweight Rep

**المبدأ:** لا يوجد %1RM. التقدم يكون بـ:
1. زيادة التكرارات
2. زيادة المجموعات
3. الانتقال لنسخة أصعب (Difficulty Ladder)

| الهدف | نطاق التكرارات | المجموعات | أولوية التقدم |
|-------|----------------|-----------|---------------|
| **STRENGTH** | 5–10 | 3–4 | difficulty ← reps ← sets |
| **HYPERTROPHY** | 10–20 | 3–5 | reps ← sets ← difficulty |
| **POWER** | 3–8 (بسرعة) | 3–4 | difficulty (نسخة انفجارية) |
| **GENERAL_HEALTH** | 10–15 | 2–3 | reps ← sets |

**مثال Difficulty Ladder:**

```
Push-up Ladder:
  1. Wall Push-up (مبتدئ)
  2. Knee Push-up
  3. Standard Push-up
  4. Diamond Push-up
  5. Archer Push-up (متقدم)
```

**الاقتراح التلقائي:**

```
آخر جلسة: Standard Push-up × 12 reps × 3 sets (formScore 85%)
الهدف: STRENGTH

→ Suggestion: Diamond Push-up × 6 reps × 3 sets
   (لأن difficulty أولاً في STRENGTH)

---

آخر جلسة: Standard Push-up × 12 reps × 3 sets (formScore 85%)
الهدف: HYPERTROPHY

→ Suggestion: Standard Push-up × 14 reps × 3 sets
   (لأن reps أولاً في HYPERTROPHY)
```

---

### النوع 3: Weighted Hold

**المبدأ:** الزمن + الوزن. لا يوجد 1RM تقليدي.

| الهدف | المدة المستهدفة | الوزن | أولوية التقدم |
|-------|----------------|-------|---------------|
| **STRENGTH** | 15–30 ثانية | عالي | load ← duration |
| **HYPERTROPHY** | 30–60 ثانية | معتدل | duration ← load |
| **GENERAL_HEALTH** | 20–45 ثانية | خفيف–معتدل | duration ← load |

**الاقتراح التلقائي:**

```
آخر جلسة: Weighted Plank × 30s × 10kg (formScore 80%)
الهدف: STRENGTH

→ Suggestion: Weighted Plank × 30s × 12.5kg
   (load أولاً — ثبّت المدة وارفع الوزن)
```

---

### النوع 4: Bodyweight Hold

**المبدأ:** الزمن + الصعوبة فقط.

| الهدف | المدة المستهدفة | أولوية التقدم |
|-------|----------------|---------------|
| **STRENGTH** | 15–30 ثانية | difficulty ← duration |
| **HYPERTROPHY** | 30–60 ثانية | duration ← difficulty |
| **GENERAL_HEALTH** | 20–60 ثانية | duration |

**مثال Difficulty Ladder:**

```
Plank Ladder:
  1. Forearm Plank (مبتدئ)
  2. Standard Plank
  3. Plank with Leg Lift
  4. RKC Plank (متقدم)
```

## الخطوة 4: بوابة الجودة (Quality Gate) — متى نسمح بالتقدم

### الشروط الأساسية (لكل الأنواع)

| الشرط | القيمة | المصدر العلمي |
|-------|--------|---------------|
| **formScore** | ≥ 70% | جودة الأداء — لا نتقدم بشكل سيء |
| **completionRate** | ≥ 85% | أكمل معظم التكرارات/المدة المطلوبة |
| **ROM** | ≥ 75% | ACSM: Full ROM يحسّن القوة |
| **Streak** | 2 جلسات متتالية | لا نتقدم من جلسة واحدة |

### شروط إضافية حسب الهدف (اختيارية — تُضاف لاحقاً)

| الشرط | متى يُستخدم | القيمة |
|-------|-------------|--------|
| **intensityPct** ≥ 75% | هدف STRENGTH فقط | لا نتقدم إذا الحمل خفيف جداً |
| **velocityLoss** ≤ 25% | هدف POWER فقط | لا نكمل إذا السرعة تراجعت كثيراً |
| **symmetry** ≥ 70% | تمارين ثنائية | لا نتقدم إذا فيه خلل واضح |

### متى نتراجع (Regression)

| الشرط | القيمة | الإجراء |
|-------|--------|---------|
| **formScore** < 55% | لجلسة واحدة | تخفيض المحور الحالي خطوة واحدة |
| **velocityLoss** > 35% | لجلستين | تخفيض الوزن أو التكرارات |

## الخطوة 6: المستخدم المحترف vs المبتدئ

### المبتدئ (الوضع الافتراضي)

- يرى: **الوزن المقترح + التكرارات + "ابدأ"**.
- لا يرى: %1RM، velocity، نسب مئوية معقدة.
- الرسائل: "أداء ممتاز — جاهز للمستوى التالي" أو "حافظ على نفس المستوى".
- **النظام موثوق — يمشي معاه بدون ما يحتاج يفهم التفاصيل.**

### المحترف (وضع متقدم — يُفعّل من Settings)

- يرى: **كل شيء** — %1RM، est1RM، velocityLoss، ROM%، tempo breakdown.
- يقدر: يعدّل الوزن، يغير نطاق التكرارات، يختار نسخة تمرين مختلفة.
- يرى: رسم بياني للتقدم عبر الوقت (est1RM trend، volume trend).

### كيف نحقق ده تقنياً

```typescript
// في Android — PostTrainingReport
if (user.displayMode === 'beginner') {
    show: suggestedWeight, repRange, formScore (كنجوم أو ✓)
    hide: est1RM number, %1RM, velocity, tempo details
} else {
    show: everything
}
```

## ملخص: المبادئ العلمية التي نمشي عليها

من ACSM 2026 — **ليس اختراع**:

1. **الحمل (Load)** أهم عامل للقوة — ≥80% 1RM.
2. **الحجم (Volume)** أهم عامل للتضخم — ≥10 مجموعات/أسبوع/عضلة.
3. **السرعة** أهم عامل للقدرة — Power RT (concentric بأقصى سرعة).
4. **ROM الكامل** يحسن القوة.
5. **الفشل ليس ضرورياً** — جهد كافٍ (RIR 2–3) يكفي.
6. **التردد** — ≥2 جلسات/أسبوع.
7. **الترتيب** — تمارين القوة الأساسية في بداية الجلسة.
8. **Periodization** ليس ضرورياً عند وجود تقدم تدريجي مناسب.
9. **نوع الجهاز** لا يهم.
10. **أي تمرين مقاومة أفضل من عدم التمرين**.
