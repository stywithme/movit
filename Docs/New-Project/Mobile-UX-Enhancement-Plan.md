# خطة تطوير تجربة المستخدم على الموبايل

> **الهدف**: تحويل التطبيق من "أداة تحليل حركة" إلى **رفيق تدريب ذكي** يأخذ المتدرب بيده من أول يوم لحد ما يوصل لهدفه.
> 
> **المبدأ**: المتدرب لا يحتاج يفهم النظام — النظام يفهمه هو.

---

## تشخيص الوضع الحالي

### ماذا يوجد الآن (28 شاشة)

| المنطقة | الشاشات | الحالة |
|---------|---------|--------|
| دخول وتسجيل | Splash → Onboarding (3 صفحات) → تسجيل/دخول | تعمل — أساسية جدًا |
| الصفحة الرئيسية | HomeFragment (إحصائيات + مستوى + برنامج + خطة اليوم + تقارير + تمارين + أكشنات) | **مزدحمة — كل شيء في مكان واحد** |
| الاختبار | PAR-Q → جلسة الاختبار → النتائج | تعمل — لكن **تبدو تقنية وجافة** |
| التدريب | ما قبل التمرين → التدريب (كاميرا) → ما بعد التمرين → التقرير (7 شاشات) | **الأقوى في التطبيق** — تحليل حقيقي |
| البرامج | قائمة → تفاصيل → يوم → جلسة → تقرير الجلسة | تعمل — لكن **ناقصة الإحساس بالتقدم** |
| المستوى | شاشة المستوى (بيانات + رادار + مناطق) | غنية بالمعلومات — لكن **صعبة الفهم** |
| الخطة | نظرة على الخطة (برامج + إعادة تقييم + تاريخ التقدم) | موجودة — لكن **لا يعرف المتدرب يوصلها بسهولة** |
| التقارير | 4 تابات (نظرة عامة + تمارين + اتجاهات + أرقام قياسية) | **ممتازة — لكن كثيفة المعلومات** |

### المشكلات الرئيسية

| المشكلة | التأثير |
|---------|--------|
| **الصفحة الرئيسية مزدحمة** | المتدرب يفتح التطبيق ولا يعرف يبدأ من فين |
| **المسار غير واضح** | لا يوجد شيء يقول للمتدرب "أنت هنا، والخطوة التالية هي..." |
| **الاختبار يبدو مخيف** | المتدرب لا يعرف ليش يعمل اختبار ولا ماذا يتوقع |
| **المستوى رقم مجرد** | المتدرب يشوف "مستوى 2" لكن لا يفهم ماذا يعني عمليًا |
| **البرنامج مجرد قائمة** | لا يوجد إحساس بالرحلة — مجرد أيام وتمارين |
| **التقدم غير محسوس** | المتدرب يتدرب أسابيع ولا يشعر بالتحسن حتى يعيد الاختبار |
| **الإشعارات غائبة** | لا يوجد شيء يرجّع المتدرب للتطبيق |
| **الداتا كثيرة** | شاشات مثل LevelProfile والتقارير مليئة بأرقام — المتدرب يحتاج قصة لا أرقام |

## الرؤية المستهدفة

### المبادئ الخمسة

| المبدأ | المعنى |
|--------|--------|
| **سؤال واحد** | في أي لحظة، المتدرب يجب أن يعرف الإجابة على: "ماذا أفعل الآن؟" |
| **تقدم ملموس** | كل يوم — حتى لو لم يتدرب — يشوف شيء يتغير |
| **قصة لا أرقام** | بدل "ROM: 72%" → "مرونة الكتف تحسنت بشكل ملحوظ" |
| **احتفال لا تقييم** | كل إنجاز — مهما كان صغير — يستحق لحظة تقدير |
| **بساطة لا تبسيط** | المعلومات العميقة موجودة لمن يريدها — لكن الواجهة الأساسية بسيطة |

### 1. إعادة تصميم رحلة المستخدم الجديد (Onboarding 2.0)

#### المشكلة الحالية
الـ Onboarding الحالي 3 صفحات عامة (AI Analysis, Camera Setup, Track Progress) ← لا يربط المتدرب بالمنصة عاطفيًا ولا يجهزه للاختبار.

#### الرؤية الجديدة

**المسار الجديد: 6 خطوات في أقل من 5 دقائق**

```
مرحبًا → تعرّف عليك → هدفك → الاختبار السريع → نتيجتك → خطتك جاهزة
```

**الخطوة 1: مرحبًا (10 ثوانٍ)**
- شاشة واحدة: "مرحبًا، أنا رفيقك في التدريب"
- رسالة بسيطة: "في 5 دقائق، هنعرف مستواك ونحدد خطتك"
- زر واحد: "يلا نبدأ"

**الخطوة 2: تعرّف عليك (30 ثانية)**
- 3-4 أسئلة سريعة (بصرية — لا حقول نصية):
  - "كم مرة تتدرب في الأسبوع؟" → أيقونات: 0 / 1-2 / 3-4 / 5+
  - "هل عندك أي ألم أو إصابة؟" → نعم / لا (لو نعم → PAR-Q+ مختصر)
  - "ما هدفك الرئيسي؟" → أيقونات: مرونة / قوة / توازن / لياقة عامة
- هذه الإجابات تؤثر على أولويات محرك الوصف لاحقًا

**الخطوة 3: هدفك (10 ثوانٍ)**
- بناءً على إجاباته — شاشة تحفيزية:
  - "هدفك: تحسين المرونة ← هنبدأ نقيّم وضعك الحالي"
- هذا يعطي المتدرب سبب واضح لعمل الاختبار

**الخطوة 4: الاختبار — لكن بشكل مختلف (3 دقائق)**
- بدل شاشة سوداء بسيطة ← تصميم مشجع:
  - **عداد بسيط**: "حركة 1 من 3"
  - **وصف بالصور/الفيديو**: فيديو قصير (3-5 ثوانٍ) يعرض الحركة الصحيحة
  - **نص تشجيعي**: "خذ وقتك — لا يوجد صح أو غلط — هذا تقييم لمعرفة نقطة البداية"
  - **بعد كل حركة**: ملاحظة سريعة لطيفة ("جميل!" / "تمام، خلينا نشوف الحركة التالية")
- في النهاية: شاشة "جاري التحليل..." مع رسوم متحركة

**الخطوة 5: نتيجتك (20 ثانية)**
- لا نعرض الأرقام الأربعة مرة واحدة
- بدلاً من ذلك — **كشف تدريجي** بأنيميشن:
  1. الـ Body Score الكبير يظهر بأنيميشن عداد (0 → النتيجة)
  2. المستوى يظهر بلون: "مستوى 2 — البناء 🔶"
  3. جملة واحدة ملخصة: "عندك أساس جيد! نقطة ضعفك الرئيسية: المرونة"
  4. زر: "شوف خطتك"

**الخطوة 6: خطتك جاهزة (10 ثوانٍ)**
- كارت البرنامج المقترح مع:
  - اسم البرنامج
  - المدة (مثلاً: 4 أسابيع)
  - "مصمم لتحسين المرونة — يناسب مستواك"
- زران: "ابدأ البرنامج" / "شوف برامج أخرى"

**الإحساس المطلوب**: المتدرب يحس إنه في أقل من 5 دقائق — النظام فهمه وجهّز له كل شيء.

### 3. نظام التقدم المرئي (Visual Progress System)

#### المشكلة
المتدرب يتدرب أسابيع ولا يشعر بالتقدم إلا بعد إعادة التقييم. هذا يقتل الحماس.

#### الحل: 4 طبقات من إظهار التقدم

**الطبقة 1: التقدم اللحظي (داخل الجلسة)**
- بعد كل تمرين في الجلسة: تقييم فوري مبسط
  - "أداؤك في القرفصاء: ممتاز ⭐"
  - "المرونة تحسنت عن آخر مرة 📈"
- بعد الجلسة كاملة: ملخص سريع
  - "أكملت 3 تمارين • متوسط الأداء 82% • أفضل من أمس بـ 4%"

**الطبقة 2: التقدم اليومي (في الصفحة الرئيسية)**
- **خط الأيام** (Streak): "3 أيام متتالية 🔥" ← يعطي إحساس بالمحافظة
- **مؤشر اليوم**: "اليوم هو يوم 12 من 28 في البرنامج" ← إحساس بالمسار
- **جملة تقدم**: "أداؤك هذا الأسبوع أفضل من الأسبوع الماضي بـ 8%" ← ملموس

**الطبقة 3: التقدم الأسبوعي (Weekly Check-in)**
- كل نهاية أسبوع — شاشة مختصرة تظهر تلقائيًا:
  - "هذا أسبوعك 📊"
  - جلسات مكتملة: 4 من 4 ✅
  - أفضل تمرين: القرفصاء (أداء 91%)
  - أكبر تحسن: مرونة الكتف +12%
  - تحدي الأسبوع القادم: "حاول تحافظ على أداء 80%+ في كل جلسة"
- يمكن مشاركة هذا الملخص كصورة

**الطبقة 4: التقدم طويل المدى (Journey Map)**
- شاشة "رحلتي" — تعرض المسار الكامل:
  ```
  ⬤ الاختبار الأولي (15 يناير)
  │  Body Score: 38 • مستوى 2
  │
  ├── برنامج المرونة (4 أسابيع)
  │   │ الأسبوع 1: ✅
  │   │ الأسبوع 2: ✅ ← أنت هنا
  │   │ الأسبوع 3: ⬜
  │   │ الأسبوع 4: ⬜
  │
  ○ إعادة تقييم متوقعة (12 فبراير)
  │
  ○ البرنامج التالي: برنامج البناء
  ```

### 4. إعادة تصميم شاشة المستوى (Level Profile 2.0)

#### المشكلة الحالية
الشاشة الحالية مليئة بالأرقام: رادار + مناطق + limiting factors + مقارنات ← **شاشة بيانات، لا شاشة مستخدم.**

#### الرؤية الجديدة: ثلاث طبقات

**الطبقة الأولى: البطاقة الشخصية (ما يراه المتدرب فورًا)**
```
┌─────────────────────────────────┐
│                                 │
│         مستوى 2                 │
│      ══ البناء ══               │
│         🔶                      │
│                                 │
│    Body Score: 52 / 100         │
│    ████████████░░░░░░░░         │
│                                 │
│    "عندك أساس جيد وبتتحسن!"    │
│                                 │
│    التالي: المستوى 3 (متوسط)    │
│    تحتاج: 13 نقطة إضافية       │
│                                 │
└─────────────────────────────────┘
```

ملاحظات:
- رقم واحد كبير (Body Score)
- شريط تقدم واضح نحو المستوى التالي
- **جملة بشرية** — لا أرقام إضافية
- هدف واضح: "كم نقطة تحتاج للمستوى التالي"

**الطبقة الثانية: نقاط القوة والضعف (سحب لأسفل)**
```
┌─────────────────────────────────┐
│  نقاط قوتك 💪                  │
│  ─────────────                  │
│  ✅ التحكم: 68 — جيد            │
│  ✅ السلامة: 72 — جيد           │
│                                 │
│  نقاط التطوير 📈                │
│  ─────────────                  │
│  ⚠️ المرونة: 38 — محدود        │
│     → البرنامج الحالي يركز عليها│
│  ⚠️ التناظر: 42 — متوسط       │
│     → بتتحسن — كانت 35 قبل     │
└─────────────────────────────────┘
```

ملاحظات:
- مقسمة إلى "قوة" و"تطوير" ← لا يوجد "ضعف"
- كل نقطة ضعف مربوطة بالعلاج: "البرنامج الحالي يعالجها"
- مقارنة بالسابق عند وجودها

**الطبقة الثالثة: التفاصيل العميقة (اختياري — زر "شوف التفاصيل")**
- الرادار التفصيلي
- خريطة الجسم الملونة
- بيانات المناطق
- فرضيات الحركة
- بوابات السلامة

> هذه الطبقة للمتدرب الفضولي أو المدرب — ليست للمتدرب العادي.

### 5. تجربة الاختبار المحسنة (Assessment 2.0)

#### المشكلة الحالية
- شاشة سوداء بسيطة بنص أبيض
- "Movement 1 of 3" ← بارد وتقني
- لا يوجد شرح كافٍ لكل حركة
- النتائج تُعرض دفعة واحدة (أرقام كثيرة)

#### الرؤية الجديدة

**قبل كل حركة:**
```
┌─────────────────────────────────┐
│   حركة 1 من 3                  │
│   ████░░░░░░░░                  │
│                                 │
│   القرفصاء العلوية              │
│   ─────────────                 │
│                                 │
│   [فيديو قصير 5 ثوانٍ]         │
│   يوضح الحركة الصحيحة          │
│                                 │
│   📋 ماذا نقيس:                │
│   • مرونة الورك والركبة        │
│   • التحكم في التوازن           │
│   • تناظر الجسم                │
│                                 │
│   💡 "خذ وقتك — المهم الحركة   │
│      الطبيعية لا السرعة"       │
│                                 │
│   [جاهز! 🚀]                   │
│                                 │
│   "أحس بألم" — تخطي            │
└─────────────────────────────────┘
```

**بين الحركات:**
```
┌─────────────────────────────────┐
│   ✅ ممتاز!                    │
│                                 │
│   نتيجة القرفصاء:              │
│   المرونة: ████████░░ 72%      │
│   التحكم: ██████████ 85%       │
│                                 │
│   "أداء ممتاز في التحكم!       │
│    المرونة محتاجة تطوير"       │
│                                 │
│   الحركة التالية: الاندفاع      │
│   [التالي →]                    │
└─────────────────────────────────┘
```

**النتيجة النهائية — كشف تدريجي:**

الشاشة 1 (2 ثوانٍ): أنيميشن عداد → Body Score

الشاشة 2 (بعد tap): المستوى + الوصف

الشاشة 3 (بعد tap): نقاط القوة والضعف (3 عناصر فقط)

الشاشة 4 (بعد tap): "خطتك جاهزة" → البرنامج المقترح

> **الإحساس المطلوب**: اكتشاف تدريجي مشوّق — ليس تقرير طبي.

### 6. تجربة التدريب اليومي (Daily Training 2.0)

#### المشكلة الحالية
ProgramSessionActivity تعرض كل الجلسات مع تفاصيل كثيرة ← المتدرب يحتاج فقط يعرف "ماذا أفعل الآن".

#### الرؤية الجديدة

**الحالة الطبيعية: شاشة "جلسة اليوم"**

شاشة مبسطة تركز على الخطوة الحالية:

```

│   جلسة الجزء السفلي            │
│   ─────────────────             │

│   ⬤ التمرين الحالي:            │
│                                 │
│   ┌─────────────────────────┐   │
│   │  🏋️ القرفصاء             │   │
│   │  3 مجموعات × 10 تكرارات │   │
│   │  الوزن: 20 كجم           │   │
```

### 7. نظام الإشعارات والتذكيرات

#### المشكلة الحالية
لا يوجد أي نظام إشعارات ← المتدرب يفتح التطبيق لو تذكّر فقط.

#### الرؤية الجديدة: إشعارات ذكية وغير مزعجة

| التوقيت | الإشعار | المثال |
|---------|---------|--------|
| **صباح يوم التدريب** | تذكير بجلسة اليوم | "صباح الخير! جلسة الجزء السفلي جاهزة لك 💪" |
| **لو ما تدرب يومين** | تشجيع لطيف | "اشتقنالك! جلسة قصيرة 20 دقيقة بتفرق 😊" |
| **بعد إكمال جلسة** | تهنئة + streak | "🔥 5 أيام متتالية! أنت ماشي صح" |
| **نهاية الأسبوع** | ملخص أسبوعي | "📊 ملخص أسبوعك جاهز — شوفه!" |
| **اقتراب إعادة التقييم** | تحضير | "أسبوع واحد على إعادة التقييم — جاهز تشوف تقدمك؟" |
| **ترقي المستوى** | احتفال | "🎉 مبروك! وصلت المستوى 3 — متوسط!" |
| **تعديل محرك التقدم** | إعلام** | "⚡ تم تعديل خطتك: زيادة الوزن في القرفصاء" |

**قواعد الإشعارات:**
- لا أكثر من إشعار واحد في اليوم (إلا بعد إكمال جلسة)
- المتدرب يقدر يتحكم في أنواع الإشعارات من الإعدادات
- الإشعارات تتوقف تمامًا لو المتدرب عطّلها

### 8. نظام الإنجازات والتحفيز (Gamification خفيف)

#### المبدأ
لا نريد تحويل التطبيق للعبة — لكن نريد **لحظات تقدير صغيرة** تحفّز المتدرب.

#### الإنجازات المقترحة

**إنجازات تلقائية (تظهر كـ popup سريع):**

| الإنجاز | الشرط | الرسالة |
|---------|-------|---------|
| 🎯 البداية | إكمال الاختبار الأولي | "خطوتك الأولى! الآن عندك خطة" |
| 🔥 أول streak | 3 أيام متتالية | "3 أيام متتالية — الانتظام هو السر!" |
| ⭐ أداء ممتاز | Form Score ≥ 90% في جلسة | "أداء ممتاز! جسمك يتعلم الحركة الصحيحة" |
| 📈 تحسن ملحوظ | أي domain يتحسن ≥ 10% | "المرونة تحسنت 10%! الشغل بان" |
| 🏆 إكمال البرنامج | إنهاء برنامج كامل | "أكملت البرنامج! حان وقت التقييم" |
| 🆙 ترقي المستوى | المستوى يرتفع بعد إعادة تقييم | "مبروك! وصلت مستوى جديد!" |
| 💪 شهر كامل | 4 أسابيع تدريب منتظم | "شهر كامل من الالتزام — بطل!" |
| 🎖️ رقم قياسي | أي personal best | "رقم قياسي جديد في القرفصاء!" |

**مبادئ التحفيز:**
- لا نقاط ولا عملات ولا تصنيفات ← فقط لحظات تقدير
- الإنجاز يظهر مرة واحدة — لا يتكرر
- لا يوجد "فشل" — فقط "لسه ما وصلت"
- التركيز على **الاستمرارية** أكثر من **الأداء**

### 12. تحسينات تجربة التدريب الحقيقي (Camera Experience)

#### تحسينات مقترحة على شاشة التدريب الحالية

**قبل التمرين:**
- بدل شاشة PreWorkout المعقدة ← شاشة مبسطة:
  - اسم التمرين + فيديو تعليمي قصير (3-5 ثوانٍ) + "ابدأ"
  - إعدادات الوزن قابلة للتعديل بسحب
  - لا حقول كثيرة — فقط الأساسي

**أثناء التمرين:**
- الشاشة الحالية ممتازة (كاميرا + تحليل + feedback فوري)
- إضافة مقترحة: **صوت تشجيعي** بعد كل set ممتاز: "أحسنت!"
- إضافة: **مؤشر تقدم الجلسة** — "Set 2 من 3 • التمرين 1 من 3"

**بعد التمرين:**
- بدل PostWorkout المعقدة ← شاشة مبسطة:
  - نجمة كبيرة + النتيجة (%)
  - جملة واحدة: "أداء ممتاز!" أو "تقدم جيد — ركز على المرونة"
  - زر: "التالي" (التمرين التالي في الجلسة) أو "انتهت الجلسة"

## نقاط للنقاش

1. **هل نعرض التمارين التكيفية للمتدرب؟**
   - الخيار أ: نعرضها بشكل طبيعي (مثل باقي التمارين)
   - الخيار ب: نشرح له إنها "تمارين إضافية بناءً على ملاحظات الحركة"
   - الخيار ج: لا نميز بينها — كلها "تمارين الاختبار"

2. **هل نعرض تعديلات محرك التقدم قبل الجلسة أم بعدها؟**
   - الخيار أ: قبل — المتدرب يعرف التعديل مقدمًا ← قد يزعجه
   - الخيار ب: بعد — يكتشف التعديل في ملخص الجلسة ← أقل إزعاج
   - الخيار ج: كلاهما — يشوف التعديل في بداية الجلسة + يُذكّر به في الملخص

3. **هل الملخص الأسبوعي إلزامي أم اختياري؟**
   - إلزامي: يظهر كـ popup عند فتح التطبيق أول مرة بعد نهاية الأسبوع
   - اختياري: يظهر كإشعار يقدر يتجاهله

4. **مستوى التفصيل في التقارير الفورية**
   - هل نعرض أرقام دقيقة (ROM: 72.3%) أم تصنيفات (جيد / ممتاز)؟
   - هل المتدرب يحتاج يشوف "مقارنة بآخر مرة" بعد كل تمرين ولا بعد الجلسة كاملة؟

5. **التعامل مع الانقطاع عن التدريب**
   - لو المتدرب انقطع أسبوع: نرسل رسالة تشجيع؟ نقترح جلسة مخففة؟
   - لو انقطع أكثر من أسبوعين: نقترح إعادة بداية البرنامج؟ نستمر من حيث توقف؟

6. **تخصيص التجربة حسب الهدف**
   - المتدرب اللي هدفه "مرونة" يشوف واجهة مختلفة عن اللي هدفه "قوة"؟
   - ولا الواجهة موحدة والاختلاف فقط في المحتوى (البرامج والتمارين)؟
