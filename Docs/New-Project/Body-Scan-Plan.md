# Body Scan — خطة التقييم الشاملة (V2)

> **تنبيه:** هذا النظام أداة wellness / fitness assessment وليس أداة تشخيص طبي. كل النتائج تُعرض كمؤشرات حركية وليست تشخيصات سريرية.

## سجل التعديلات

### V2 (الحالية)

- إضافة تنبيه واضح أن النظام `wellness/fitness assessment` وليس تشخيصا طبيا.
- إعادة صياغة الرؤية لتجنب أي وعود مبالغ فيها، والتركيز على أن النتائج ملف مبدئي مع حدود دقة معروفة.
- اعتماد هيكل ثلاثي الطبقات بشكل صريح: `Measurement` ثم `Interpretation` ثم `Prescription`.
- إدخال نموذج الثقة (`HIGH/MEDIUM/LOW`) وربطه بقواعد القياس (visibility, SD, occlusion) وتأثيره على الحكم.
- توضيح قاعدة `Inconclusive` عند انخفاض الثقة بدل فرض تقييم غير موثوق.
- استبدال لغة "تشخيص" بلغة "فرضيات" مع تأكيد أنها عوامل محتملة وليست أحكام سريرية.
- إضافة فحص ما قبل الاختبار `PAR-Q+` كشرط أمان قبل البدء.
- فصل سياسات الأمان في طبقة التوصيات عبر `Safety Gates` على مستوى المنطقة (regional gating).
- تحديث فلسفة القياس لتأكيد قاعدة "لا نقيس ما لا نراه" ومنع الاستنتاجات من زوايا تصوير غير مناسبة.
- توسيع الخطة لتشمل عناصر تنفيذية أوضح (المعايير المرجعية، واجهات النتائج، الربط بالتوصيات، مراحل التنفيذ).

### V1 (النسخة الأولية)

- إنشاء مستند الخطة الأساسية للـ Body Scan وهيكل المشروع العام.

## الرؤية

الـ Body Scan هو **أول تفاعل حقيقي بين التطبيق وجسم المتدرب**. بدلاً من سؤال المستخدم "ما مستواك؟" — نقيس بأنفسنا. في جلسة قصيرة، الكاميرا بتبني **ملف مبدئي** عن قدرات الجسم الحركية: المرونة، التوازن، التماثل، جودة الحركة. هذا الملف يُستخدم كنقطة انطلاق لتخصيص التدريب — مع إدراك أن القياس بالكاميرا له حدود دقة معروفة.

---

## المبدأ الأساسي: لا نبني نظام جديد

الـ Body Scan **ليس** نظام منفصل عن نظام التمارين. هو **استخدام ذكي للبنية التحتية الموجودة** مع إضافة طبقة تفسير فوقها.

| الطبقة | موجودة؟ | الدور |
|---|---|---|
| TrainingEngine (قياس الزوايا والحركة) | موجودة بالكامل | تشغيل الحركات وقياس الأداء |
| MotionRecorder (تسجيل كل frame) | موجودة بالكامل | تسجيل البيانات الخام |
| MetricsCalculator (ROM, Stability, Symmetry...) | موجودة بالكامل | حساب المقاييس |
| ReportGenerator (تقرير لكل تمرين) | موجودة بالكامل | إنتاج تقارير الأداء |
| WorkoutConfig (سلسلة تمارين) | موجودة بالكامل | إدارة جلسة متعددة التمارين |
| **AssessmentEngine** (تفسير النتائج) | **جديدة** | تحويل التقارير لـ Body Profile |
| **Reference Norms** (معايير مرجعية) | **جديدة** | مقارنة المتدرب بالمعايير |
| **Body Profile** (ملف القدرات) | **جديد** | حفظ وتتبع قدرات المتدرب |
| **Assessment UI** (شاشات النتائج) | **جديدة** | عرض النتائج والتوصيات |

**ما هو جاهز فعلاً وما يحتاج عمل:**
البنية التحتية للقياس والتسجيل جاهزة. لكن طبقة التفسير (تحويل زوايا لتقييمات)، وجدول المعايير المرجعية، وشاشات العرض، والربط بالتوصيات — كل ده عمل جديد. الجزء الجاهز يوفّر علينا بناء محرك القياس من الصفر، لكنه ليس "90% من المشروع".

## الهيكل ثلاثي الطبقات — Three-Layer Architecture

كل قرار في النظام يمر بـ 3 طبقات منفصلة بوضوح:

### الطبقة 1: القياس (Measurement)

- **المدخل:** بيانات خام من TrainingEngine (زوايا، مواقع landmarks، timestamps)
- **المخرج:** أرقام + مستوى ثقة لكل رقم
- **القاعدة:** لا تفسير هنا — فقط أرقام وثقة

```
مثال المخرج:
┌─────────────────────────────────────────────────┐
│ Hip Flexion ROM: 95°                            │
│ Confidence: HIGH (visibility > 0.85, SD < 3°)  │
│ Measurement Error Band: ±5°  (range: 90°–100°) │
└─────────────────────────────────────────────────┘
```

**مستويات الثقة:**

| المستوى | الشروط | الإجراء |
|---|---|---|
| HIGH | visibility > 0.85 لكل landmark مطلوب + SD < 5° عبر التكرارات | النتيجة موثوقة — تُستخدم في التقييم |
| MEDIUM | visibility 0.70–0.85 أو SD 5°–10° | النتيجة تقديرية — تُعرض مع تنبيه |
| LOW | visibility < 0.70 أو SD > 10° أو occlusion ملحوظ | **"يحتاج إعادة"** — لا تُستخدم في التقييم |

### الطبقة 2: التفسير (Interpretation)

- **المدخل:** أرقام القياس + مستوى الثقة
- **المخرج:** تقييم المنطقة + فرضيات (وليس تشخيصات)
- **القاعدة:** إذا القياس LOW → المنطقة = "غير مؤكد" (Inconclusive) — لا يتم تقييمها

```
مثال المخرج:
┌─────────────────────────────────────────────────────────┐
│ Hip Mobility: 79% of reference norm                     │
│ Status: GOOD                                            │
│ Confidence: HIGH                                        │
│                                                         │
│ Observed compensation: Excessive trunk lean              │
│ Possible factors (not diagnosis):                       │
│   → Tight hip flexors (likely)                          │
│   → Weak core stabilizers (possible)                    │
│   → Limited ankle dorsiflexion (possible)               │
│                                                         │
│ Suggested confirmation: Forward Fold depth check ✓ Done │
│ Suggested confirmation: Ankle ROM check ✓ Done          │
└─────────────────────────────────────────────────────────┘
```

### الطبقة 3: التوصيات (Prescription)

- **المدخل:** تقييمات المناطق + الفرضيات المؤكدة
- **المخرج:** قرارات تدريبية محددة
- **القاعدة: Safety Gates** — أي منطقة حمراء **توقف التصعيد** في التمارين المرتبطة بها حتى لو Body Score جيد

```
مثال:
┌─────────────────────────────────────────────────────────────┐
│ ⛔ SAFETY GATE: Ankle Mobility = LIMITED (38%)              │
│ → Block: Full Squat, Pistol Squat, Jump Squat              │
│ → Allow: Box Squat (heel elevated), Leg Press              │
│ → Priority Fix: Ankle mobility program (4-6 weeks)         │
│                                                             │
│ ✅ CLEAR: Shoulder Mobility = GOOD (78%)                    │
│ → Allow: Overhead Press, Pull-ups                           │
│ → Suggestion: Add shoulder warm-up routine                  │
└─────────────────────────────────────────────────────────────┘
```

## فحص ما قبل الاختبار — Pre-Screening (PAR-Q+)

قبل أي اختبار بدني، المستخدم يجب أن يجتاز فحص سلامة مبسّط (مستوحى من PAR-Q+ المعتمد دولياً).

### الأسئلة (7 أسئلة — نعم / لا):

1. هل أخبرك طبيب بأن لديك مشكلة في القلب؟
2. هل تشعر بألم في الصدر أثناء النشاط البدني؟
3. هل فقدت التوازن بسبب دوخة أو فقدت الوعي مؤخراً؟
4. هل لديك مشكلة في العظام أو المفاصل قد تتأثر بالتمارين؟
5. هل يصف لك طبيب أدوية لضغط الدم أو القلب حالياً؟
6. هل هناك أي سبب آخر يمنعك من ممارسة النشاط البدني؟
7. هل أنتِ حامل أو ولدتِ خلال الأشهر الـ 6 الأخيرة؟

### المنطق:

| النتيجة | الإجراء |
|---|---|
| كل الإجابات "لا" | ✅ ابدأ الاختبار |
| إجابة واحدة أو أكثر "نعم" | ⚠️ رسالة: "ننصحك باستشارة طبيبك قبل البدء. يمكنك المتابعة على مسؤوليتك." + checkbox موافقة |

**ملاحظة:** هذا ليس فحص طبي — هو حد أدنى من الحماية القانونية والأخلاقية.

## بطارية الاختبار

### فلسفة اختيار الحركات

كل حركة في الاختبار مختارة لتحقق 3 شروط:
1. **تفحص منطقة محددة** من الجسم
2. **قابلة للقياس من زاوية تصوير واحدة** (كل المقاييس المطلوبة مرئية من نفس الزاوية)
3. **آمنة للجميع** (بدون أوزان، بدون حركات خطرة)

### قاعدة ذهبية: لا نقيس ما لا نراه

> **قياس Knee Valgus يحتاج كاميرا أمامية (frontal plane). قياس ROM يحتاج كاميرا جانبية (sagittal). لا نخلط بينهم.**
>
> إذا كانت زاوية التصوير جانبية، فإن أي مقياس يعتمد على المستوى الأمامي (مثل Valgus) لا يُقاس — بدلاً من تقديم رقم مضلل.

---

### الحركات الأساسية (MVP — 3 حركات أساسية)

نبدأ بـ 3 حركات عالية القيمة بدلاً من 5، لتقليل مدة الاختبار وزيادة الموثوقية. حركات إضافية تُضاف لاحقاً عند ظهور flags تحتاج تأكيد (Adaptive Battery).

---

#### 1. Overhead Squat — القرفصاء مع رفع الذراعين

- **النوع:** Rep (5 تكرارات)
- **Tempo:** إرشادي وليس إجباري (توجيه: "بإيقاع هادي ومريح")
- **المناطق:** ورك + كاحل + كتف + جذع
- **زاوية التصوير:** جانبية (Sagittal) — **فقط مقاييس المستوى الجانبي**

**ما نقيسه (من المنظر الجانبي فقط):**

| المقياس | التفصيل | قابلية القياس |
|---|---|---|
| ROM الورك (Flexion) | أقصى زاوية ثني وصل لها | عالية — landmarks واضحة |
| ROM الكاحل (Dorsiflexion) | زاوية الكاحل أثناء النزول | متوسطة — يعتمد على رؤية القدم |
| ROM الكتف (Flexion) | هل الذراعين بتفضل فوق الرأس | عالية — الكتف واضح جانبياً |
| Trunk Alignment | زاوية ميل الجذع عن الرأسي | عالية — Shoulder-Hip-Knee واضحين |
| Squat Depth | هل الفخذ وصل لمستوى الركبة أو تحتها | عالية |
| Stability | ثبات مركز الجسم (trunk sway) | عالية |

**ما لا نقيسه من هذه الزاوية:**
- ~~Knee Valgus~~ (يحتاج منظر أمامي — لا نقيسه هنا)
- ~~Foot pronation~~ (يحتاج كاميرا قريبة — خارج النطاق)

**التعويضات الحركية المرصودة (Hypothesis — ليست تشخيص):**

| الملاحظة | الأسباب المحتملة (فرضيات) |
|---|---|
| سقوط الذراعين للأمام | شد في الصدر / ضعف أعلى الظهر / محدودية الكتف |
| ميل الجذع المفرط للأمام | محدودية الكاحل / ضعف Core / شد في الورك |
| عدم الوصول لعمق كافي | قيد في الورك / قيد في الكاحل / ضعف عام |
| ارتفاع الكعب | قصر عضلات السمانة / محدودية الكاحل |

**لماذا هذه الحركة؟**
الـ Overhead Squat هو الاختبار الأشمل (معتمد من NASM كأداة فحص). حركة واحدة ترصد مؤشرات عن السلسلة الحركية من الكاحل للكتف.

---

#### 2. Lunge — الطعن

- **النوع:** Rep (3 تكرارات لكل جانب)
- **الوضع:** Bilateral Alternating
- **Tempo:** إرشادي ("بإيقاع مريح")
- **المناطق:** ورك + ركبة + توازن ديناميكي
- **زاوية التصوير:** جانبية

**ما نقيسه:**

| المقياس | التفصيل | قابلية القياس |
|---|---|---|
| ROM الورك الأمامي | عمق النزول (زاوية الركبة الأمامية) | عالية |
| ROM الورك الخلفي | مد الورك للساق الخلفية | عالية |
| Symmetry L/R | LSI بين الساقين | عالية (bilateral) |
| Balance/Stability | ثبات الجذع أثناء الحركة (trunk sway) | عالية |
| Form Consistency | انتظام التكرارات | عالية |

**التعويضات الحركية المرصودة:**

| الملاحظة | الأسباب المحتملة |
|---|---|
| ميل الجذع للأمام | ضعف Core / شد في الورك الخلفي |
| عمق غير كافي (ركبة > 90°) | قصر قابضات الورك / ضعف عام |
| اهتزاز ملحوظ | ضعف التوازن / ضعف Glutes |
| فرق كبير بين الجانبين | عدم تماثل — يحتاج اهتمام |

**لماذا هذه الحركة؟**
الـ Lunge يكشف عدم التماثل بين الساقين (مشكلة شائعة جداً) ويختبر التوازن الديناميكي مع المرونة في آن واحد. وبصفته Bilateral Alternating، النظام الموجود يقيس LSI تلقائياً.

---

#### 3. Shoulder Mobility Test — اختبار مرونة الكتف

- **النوع:** Rep (5 تكرارات لكل جانب)
- **الوضع:** Bilateral Alternating
- **المناطق:** كتف + أعلى الظهر
- **زاوية التصوير:** جانبية

**ما نقيسه:**

| المقياس | التفصيل | قابلية القياس |
|---|---|---|
| ROM الكتف (Flexion) | أقصى رفع للذراع | عالية — واضح جانبياً |
| Symmetry L/R | LSI بين الكتفين | عالية (bilateral) |
| Movement Quality | سلاسة الحركة (Form Consistency) | عالية |

**ما لا نقيسه من هذه الزاوية:**
- ~~Abduction~~ (يحتاج منظر أمامي)
- ~~External Rotation~~ (يحتاج وضع خاص غير عملي)

**التعويضات الحركية المرصودة:**

| الملاحظة | الأسباب المحتملة |
|---|---|
| تقوّس الظهر أثناء الرفع | شد في الصدر / ضعف Core |
| عدم الوصول لـ 170°+ | محدودية في مفصل الكتف / شد عضلي |
| فرق > 15° بين الجانبين | عدم تماثل ملحوظ — قد يحتاج اهتمام |

**لماذا هذه الحركة؟**
الكتف أكثر مفصل يتعرض لمشاكل حركية (خصوصاً عند من يعملون مكتبياً). التماثل بين الجانبين مؤشر مهم على صحة المفصل.

---

### الحركات الإضافية (تُفعّل عند الحاجة — Adaptive Battery)

هذه الحركات لا تظهر تلقائياً. تُضاف فقط عندما ترصد الحركات الأساسية flags تحتاج تأكيد أو تعمّق.

#### 4. Forward Fold — الانحناء الأمامي (يُفعّل تلقائياً)

**يُفعّل عندما:** ميل جذع مفرط في Overhead Squat أو عمق محدود
- **النوع:** Hold (10 ثواني)
- **الهدف:** تأكيد: هل المشكلة في خلفية الفخذ (Hamstrings) أم في الورك/الكاحل؟
- **ما نقيسه:** ROM ثني الجذع، Spine Curvature، Hold Stability

#### 5. Single-Leg Balance — التوازن على رجل واحدة (يُفعّل تلقائياً)

**يُفعّل عندما:** ضعف stability ملحوظ في Lunge أو فرق تماثل كبير
- **النوع:** Hold (15 ثانية لكل جانب)
- **الهدف:** عزل مشكلة التوازن — هل هي عامة أم في جانب واحد
- **ما نقيسه:** Hold Duration، Stability Sway، Symmetry L/R

#### قواعد التفعيل:

| Flag من الحركات الأساسية | الحركة المضافة | السبب |
|---|---|---|
| Trunk lean > threshold في Squat | Forward Fold | تأكيد: hamstrings أم ankle |
| Stability score < 60% في Lunge | Single-Leg Balance | عزل: توازن عام أم جانب واحد |
| LSI < 80% في Lunge | Single-Leg Balance | تأكيد: التوازن أم ROM الفرق |
| Squat depth محدود + ankle ROM منخفض | Forward Fold | تأكيد: المشكلة في hip flexion أم ankle |

## نظام القياس

### المقاييس المُستخرجة: نفسها مع إضافات

المقاييس الموجودة حالياً في النظام تعمل كما هي. لكن الاختبار يتطلب **بيانات إضافية**:

#### ما هو موجود ويُستخدم كما هو

- **Rep Score** — جودة كل تكرار
- **ROM** (كنسبة من الـ target)
- **Symmetry** (LSI للتمارين الثنائية)
- **Stability** (ثبات الجذع)
- **Alignment Accuracy** (Position Checks)
- **Form Consistency** (اتساق الأداء عبر التكرارات)

#### ما هو جديد ومطلوب للاختبار

| المقياس الجديد | الوصف | لماذا مطلوب |
|---|---|---|
| **Absolute ROM (بالدرجات)** | القيمة المطلقة لأقصى وأدنى زاوية + Error Band (±) | للمقارنة مع المعايير المرجعية مع مراعاة هامش الخطأ |
| **ROM vs Norm (%)** | نسبة ROM المتدرب مقارنة بالمعيار المرجعي | لتحديد مستوى المرونة (مع تصنيف الثقة) |
| **Movement Quality Grade** | 1 = فشل / محدود، 2 = تعويض ملحوظ، 3 = حركة نظيفة | بناءً على عدد/نوع التعويضات المرصودة |
| **Pain Flag** | علم منفصل تماماً عن التقييم: هل بلّغ المتدرب عن ألم | يوقف التقييم فوراً — ليس درجة، بل حالة |
| **Compensation Hypotheses** | قائمة بالملاحظات الحركية + الأسباب المحتملة | فرضيات قابلة للتأكيد — وليست تشخيصات |
| **Confidence Level** | HIGH / MEDIUM / LOW لكل قياس | لتحديد موثوقية النتيجة |
| **Regional Score** | درجة لكل منطقة من الجسم (0-100) + مستوى الثقة | لبناء خريطة الجسم |

### Measurement Error — حدود الدقة المعروفة

> **مهم:** القياس بكاميرا واحدة (2D markerless) له حدود دقة موثّقة علمياً. نحن نعترف بها ونتعامل معها بشفافية.

| المفصل | هامش الخطأ المتوقع | المصدر |
|---|---|---|
| الكتف (Flexion) | ±5°–7° | Pose estimation benchmarks |
| الورك (Flexion) | ±5°–8° | Pose estimation benchmarks |
| الركبة (Flexion) | ±4°–6° | Pose estimation benchmarks |
| الكاحل (Dorsiflexion) | ±8°–12° | أقل موثوقية — landmark صغير |
| العمود الفقري | ±6°–10° | يعتمد على الملابس والرؤية |

**تطبيق عملي:**
- القياس 95° بهامش ±7° = النطاق الحقيقي **88°–102°**
- إذا المعيار المرجعي 110° → النتيجة تتراوح بين **80%–93%** من المعيار
- النظام يعرض **القيمة المركزية + النطاق** ولا يعطي رقم واحد قاطع
- **قاعدة التقدم:** لا نعتبر أن تحسن حصل إلا إذا الفرق بين الاختبارين > MDC (Minimal Detectable Change ≈ هامش الخطأ × 1.5)

### جدول المعايير المرجعية (Reference Norms)

> هذه معايير مرجعية عامة للبالغين الأصحاء (18-45 سنة). ليست معايير طبية تشخيصية. يمكن تخصيصها لاحقاً حسب العمر والجنس.

| المنطقة | الحركة | ROM المرجعي | ملاحظة |
|---|---|---|---|
| **الكتف** | Flexion (رفع أمامي) | 150°–180° | AAOS Guidelines |
| **الورك** | Flexion (ثني) | 110°–120° | AAOS |
| **الورك** | Extension (مد) | 20°–30° | AAOS |
| **الركبة** | Flexion (ثني) | 130°–140° | AAOS |
| **الكاحل** | Dorsiflexion | 15°–20° | أقل موثوقية بالكاميرا |
| **العمود الفقري** | Flexion (انحناء أمامي) | 75°–90° | AMA Guides |

**ملاحظة عن الكاحل:** بسبب هامش الخطأ العالي (±8-12°)، قياس الكاحل يُعامل بحذر. يُستخدم كمؤشر عام فقط وليس كقياس دقيق.

## نظام التقييم

### Domain Scores — بدلاً من رقم واحد

بدلاً من Body Score واحد يخفي التفاصيل، النظام يُنتج **4 أبعاد مستقلة** (مستوحاة من نفس فلسفة Form/Safety/Control في تقرير التمرين):

| البُعد | ما يقيسه | المصادر |
|---|---|---|
| **Mobility** (المرونة) | ROM لكل منطقة مقارنة بالمعيار | Absolute ROM من كل الحركات |
| **Control** (التحكم) | جودة الحركة + الثبات | Stability + Form Consistency + Movement Quality |
| **Symmetry** (التماثل) | التوازن بين الجانبين | LSI من Lunge + Shoulder Test |
| **Safety** (السلامة) | عدد ونوع التعويضات + Pain flags | Compensation count + severity |

### حساب كل بُعد (0–100)

#### Mobility Score

```
لكل منطقة:
  ROM_ratio = Measured_ROM / Reference_Norm
  clamp(ROM_ratio × 100, 0, 100)

Mobility = المتوسط المرجّح للمناطق (weighted by measurement confidence)
```

- القياسات ذات ثقة LOW → تُستبعد من المتوسط
- القياسات ذات ثقة MEDIUM → وزنها نصف وزن HIGH

#### Control Score

```
Control = (Stability × 0.4) + (Form_Consistency × 0.35) + (Movement_Quality_Grade × 0.25)

حيث Movement_Quality_Grade: 3=100, 2=60, 1=20
```

#### Symmetry Score

```
Symmetry = متوسط LSI لكل المناطق الثنائية

LSI = (الجانب الأضعف / الجانب الأقوى) × 100
```

- إذا لم توجد تمارين ثنائية → لا يُعرض هذا البُعد

#### Safety Score

```
Safety = 100 - (compensation_penalties)

حيث:
  كل تعويض WARNING = -10
  كل تعويض DANGER = -25
  Pain flag = 0 (توقف فوري، لا يُحسب كـ score)
```

### Safety Gates — قواعد الحجب

> **القاعدة الأهم:** أي منطقة تحصل على تصنيف "محدود" أو "ضعيف" **تمنع تصعيد التمارين** المرتبطة بها، بغض النظر عن أي Score آخر.

| الحالة | القرار | لا يمكن تجاوزه |
|---|---|---|
| أي Mobility < 30% لمنطقة | حجب التمارين المتقدمة لهذه المنطقة | نعم |
| Pain Flag في أي حركة | توقف + توصية بفحص متخصص | نعم |
| Safety Score < 40% | برنامج تصحيحي فقط — لا تدريب قوة | نعم |
| Symmetry < 75% لمنطقة | أولوية تمارين أحادية الجانب | نعم |
| أي قياس بثقة LOW | المنطقة = "يحتاج إعادة" — لا يُبنى عليها قرار | نعم |

### Body Score — كملخص عام فقط

الـ Body Score **يظل موجود** كرقم تحفيزي سريع، لكنه **لا يُستخدم في اتخاذ قرارات تدريبية**. القرارات تُبنى على Domain Scores + Safety Gates.

```
Body Score = (Mobility × 0.35) + (Control × 0.25) + (Symmetry × 0.20) + (Safety × 0.20)
```

| Body Score | المستوى | الوصف |
|---|---|---|
| 85+ | ممتاز | مؤشرات حركية قوية |
| 65–84 | جيد | مستوى صحي — بعض المناطق تحتاج اهتمام |
| 45–64 | متوسط | قيود ملحوظة — برنامج تصحيحي مُوصى |
| 25–44 | محدود | قيود مؤثرة — التركيز على التصحيح أولاً |
| 0–24 | يحتاج تأهيل | يُنصح باستشارة متخصص |

### تصنيف المناطق

| المستوى | النسبة | اللون | الوصف |
|---|---|---|---|
| ممتاز | 85–100% | أخضر | لا قيود ملحوظة |
| جيد | 65–84% | أخضر فاتح | مجال بسيط للتحسن |
| متوسط | 45–64% | أصفر | قيود ملحوظة — يحتاج عمل |
| محدود | 25–44% | برتقالي | قيود مؤثرة — **Safety Gate** يُفعّل |
| ضعيف | 0–24% | أحمر | قيود شديدة — **Safety Gate** + توصية متخصص |

## نظام الفرضيات الحركية — Hypothesis Cards

### الفلسفة: ملاحظة + فرضيات + تأكيد

بدلاً من "عندك شد في الصدر" (ادعاء تشخيصي)، النظام يقول:

```
┌──────────────────────────────────────────────────┐
│ 📋 ملاحظة حركية                                  │
│                                                    │
│ "سقوط الذراعين للأمام أثناء القرفصاء"            │
│                                                    │
│ الأسباب المحتملة:                                 │
│  1. محدودية مرونة الكتف (مؤكد ✅ — ROM 72%)       │
│  2. شد في عضلات الصدر (محتمل — يحتاج تأكيد)     │
│  3. ضعف أعلى الظهر (محتمل — يحتاج تأكيد)        │
│                                                    │
│ التوصية:                                           │
│  → تمارين مرونة الكتف (أولوية عالية)              │
│  → تمارين تقوية أعلى الظهر (أولوية متوسطة)       │
│                                                    │
│ 📊 مستوى الثقة: HIGH                              │
└──────────────────────────────────────────────────┘
```

### كيف يعمل التأكيد التلقائي

بما أن البطارية تحتوي على أكثر من حركة، بعض الفرضيات يمكن **تأكيدها أو نفيها تلقائياً** بمقارنة النتائج:

| الفرضية | حركة التأكيد | كيف |
|---|---|---|
| محدودية الكاحل تسبب ميل الجذع | Overhead Squat ankle ROM | إذا ankle ROM منخفض → الفرضية مؤكدة |
| ضعف التوازن عام | Single-Leg Balance (إن فُعّل) | إذا ضعيف في الجانبين → عام |
| عدم التماثل في الورك | Lunge LSI | إذا LSI < 80% → مؤكد |
| محدودية الكتف | Shoulder Test ROM | تأكيد مباشر |

## خريطة الجسم البصرية — Body Map

### الفكرة

شاشة فيها رسم توضيحي لجسم الإنسان. كل منطقة ملوّنة حسب تقييمها. المتدرب يشوف في لحظة واحدة: فين نقاط القوة وفين نقاط الضعف.

### المناطق المعروضة على الخريطة

| المنطقة | الجانب | مصدر التقييم |
|---|---|---|
| الكتف | أيمن + أيسر | Shoulder Mobility Test |
| أعلى الظهر | وسط | Overhead Squat + Shoulder Test |
| الجذع / Core | وسط | Stability من كل الحركات |
| أسفل الظهر | وسط | Forward Fold (إن فُعّل) + Overhead Squat |
| الورك | أيمن + أيسر | Overhead Squat + Lunge |
| الركبة | أيمن + أيسر | Lunge |
| الكاحل | أيمن + أيسر | Overhead Squat (بثقة أقل) |

### عرض الثقة في الخريطة

كل منطقة تظهر بلونها **+ شارة ثقة صغيرة**:
- ثقة HIGH → اللون ثابت
- ثقة MEDIUM → اللون بتدرج مخفف + أيقونة "~"
- ثقة LOW → رمادي + علامة "?" (يحتاج إعادة)

### التفاعل

- الضغط على أي منطقة يعرض: ROM الفعلي vs المعيار (مع هامش الخطأ)، التعويضات المرصودة، التوصيات
- اللون يتغير مع إعادة الاختبار (تقدم مرئي)

## بروتوكول الاختبار — Protocol Design

### مسألة الـ Tempo: إرشادي وليس إجباري

**المشكلة السابقة:** الخطة كانت تقول "لا نصحح" و"Tempo إجباري" — وهذا تناقض. الإيقاع الإجباري هو نفسه شكل من أشكال التوجيه.

**الحل:**

| الحركة | النهج |
|---|---|
| حركات ROM (Squat, Lunge, Shoulder) | **Tempo إرشادي** — توجيه لطيف ("بإيقاع هادي ومريح") بدون enforcement صارم. السبب: نريد أن نرصد الحركة الطبيعية للشخص |
| حركات Hold (Forward Fold, Balance) | **لا tempo** — فقط عداد تنازلي للمدة |

**الـ Feedback أثناء الاختبار:**

| مسموح | غير مسموح |
|---|---|
| عد تكرارات / عد تنازلي | "ارفع صدرك" / "ركبتك داخل" |
| "ممتاز، كمّل" / "فاضل 3" | أي تصحيح للشكل |
| "خد وقتك، بإيقاع مريح" | تنبيهات Position Check |

**ملاحظة:** Position Checks **تعمل** في الخلفية وتُسجّل — لكن لا تظهر كـ feedback للمستخدم أثناء الاختبار.

## مواعيد الاختبار — Assessment Schedule

### الاختبار الأولي (Initial Assessment)

- **متى:** أول استخدام للتطبيق (جزء من الـ Onboarding)
- **الهدف:** بناء الـ Baseline
- **الرسالة:** "قبل ما نبدأ، خلينا نتعرف على حركتك — دقائق بسيطة بدون أي معدات"

### الاختبار الدوري (Periodic Re-test)

- **متى:** كل 4-6 أسابيع (أو بعد نهاية كل مرحلة تدريبية)
- **الهدف:** قياس التقدم
- **الأهمية:** **نفس الحركات بالظبط** — لتكون المقارنة عادلة
- **قاعدة التقدم:** لا نحتفل بتحسن إلا إذا الفرق > MDC (Minimal Detectable Change)

### اختبار ما بعد البرنامج (Post-Program Assessment)

- **متى:** بعد إنهاء أي برنامج تدريبي
- **الهدف:** قياس تأثير البرنامج بشكل موضوعي

### شكل التقدم عبر الزمن

```
يناير (Baseline)          فبراير (Re-test)          مارس (Post-Program)
───────────────────      ───────────────────      ───────────────────
Mobility:  52 (متوسط)    Mobility:  58 (~)         Mobility:  67 (جيد ✅)
Control:   61 (متوسط)    Control:   65 (جيد ✅)    Control:   71 (جيد)
Symmetry:  78 (جيد)      Symmetry:  82 (جيد)      Symmetry:  85 (ممتاز ✅)
Safety:    45 (⚠️ Gate)   Safety:    62 (متوسط)    Safety:    78 (جيد ✅)

Body Score: 56            Body Score: 64            Body Score: 74

(~) = التغيير أقل من MDC — لا نعتبره تحسن حقيقي
(✅) = تحسن حقيقي تجاوز MDC
```

## من الاختبار إلى البرنامج — Recommendations Engine

### المنطق الأساسي

نتائج الاختبار بتُترجم لقرارات تدريبية عبر 4 مراحل (Corrective Exercise Continuum — NASM):

### المرحلة 1: التثبيط (Inhibit)
- **الهدف:** إرخاء العضلات المشدودة
- **مثال:** لو الفرضية تشير لشد في الصدر → Foam Rolling للصدر

### المرحلة 2: الإطالة (Lengthen)
- **الهدف:** زيادة مرونة المناطق المحدودة
- **مثال:** لو الكاحل محدود → Static Stretching للسمانة

### المرحلة 3: التنشيط (Activate)
- **الهدف:** تقوية العضلات الضعيفة (الفرضيات المؤكدة)
- **مثال:** لو ضعف Core مؤكد → تمارين عزل Core

### المرحلة 4: الدمج (Integrate)
- **الهدف:** تطبيق الحركة الصحيحة في سياق مركّب
- **مثال:** بعد التصحيح → Squat بعمق محسّن

### ترجمة النتائج لقرارات (مع Safety Gates)

| نتيجة الاختبار | القرار التدريبي | Safety Gate |
|---|---|---|
| Ankle Mobility < 40% (+ ثقة HIGH) | لا Squat كامل — Box Squat مع رفع الكعب | **حجب** Deep Squat حتى > 50% |
| LSI الورك < 80% | تمارين أحادية الجانب — جرعة أكبر للجانب الأضعف | **أولوية** لا حجب |
| Stability < 40% في Balance | برنامج توازن قبل أي تمرين مركّب | **حجب** plyometrics |
| Shoulder Mobility < 50% (+ ثقة HIGH) | لا Overhead Press — Landmine Press كبديل | **حجب** overhead movements |
| Safety Score < 40% | برنامج تصحيحي 4-6 أسابيع قبل تدريب القوة المتقدم | **حجب** تمارين القوة المتقدمة |
| Pain Flag | **توقف فوري** + "ننصحك باستشارة متخصص" | **حجب كامل** للمنطقة |
| قياس بثقة LOW | المنطقة = "غير مؤكد" — لا يُبنى عليها قرار | **إعادة** القياس |

**ملاحظة مهمة:** كل قرار مبني على ثقة HIGH أو MEDIUM. القياسات بثقة LOW لا تُنتج قرارات — فقط طلب إعادة.

## تدفق تجربة المستخدم — User Flow

### الاختبار الأولي (Onboarding)

```
1. PAR-Q+ (فحص ما قبل الاختبار)
   7 أسئلة بسيطة — نعم/لا
   ✅ كل الإجابات "لا" → التالي
   ⚠️ أي "نعم" → رسالة + خيار المتابعة بموافقة

2. شاشة ترحيب
   "خلينا نتعرف على حركتك"
   "اختبار بسيط — بدون أي معدات"
   [ابدأ الآن]

3. إعداد الكاميرا
   "ضع الهاتف على مسافة 2-3 متر"
   "تأكد إن جسمك كامل ظاهر"
   [معاينة مباشرة مع إطار إرشادي]
   ✅ جودة الرؤية كافية → [التالي]
   ❌ جودة ضعيفة → إرشادات تحسين

4. لكل حركة (3 أساسية + حركات إضافية إن لزم):
   أ. شاشة شرح الحركة
      - اسم الحركة + فيديو توضيحي (5-10 ثواني)
      - "المطلوب: 5 تكرارات بإيقاع مريح"
      - [أحس بألم في هذه الحركة — تخطي] ← Pain Flag
      [جاهز]
   
   ب. عد تنازلي (3-2-1)
   
   ج. أداء الحركة
      - TrainingEngine يعمل في الخلفية
      - عداد التكرارات ظاهر
      - رسائل تشجيعية فقط (لا تصحيح)
      - زرار "أحس بألم" ← إيقاف فوري
   
   د. فحص الجودة (تلقائي):
      - ثقة HIGH → ✅ "ممتاز! الحركة التالية..."
      - ثقة LOW → "الرؤية كانت ضعيفة — نعيد؟" [إعادة / تخطي]
   
   هـ. (فحص Adaptive): هل ظهرت flags تحتاج تأكيد؟
      - نعم → "عندنا حركة إضافية قصيرة للتأكد..." (Forward Fold أو Balance)
      - لا → الحركة التالية

5. شاشة المعالجة
   "جاري تحليل حركتك..."
   (2-3 ثواني)

6. شاشة النتائج
   أ. Body Map (خريطة الجسم الملوّنة + شارات الثقة)
   ب. Domain Scores (Mobility, Control, Symmetry, Safety)
   ج. Body Score (كرقم تحفيزي)
   د. أبرز 2-3 نقاط (أقوى + أضعف)
   هـ. Hypothesis Cards (أبرز ملاحظة حركية + الأسباب المحتملة)
   و. [شارك نتيجتك] [ابدأ برنامجك المخصص]
```

### إعادة الاختبار

```
1. إشعار / اقتراح
   "مر شهر من آخر اختبار — يلا نشوف تقدمك!"

2. نفس الخطوات (1-5)

3. شاشة المقارنة
   أ. Domain Scores: قبل → بعد (مع تمييز التحسن الحقيقي > MDC)
   ب. Body Map: جنب بعض (قبل وبعد)
   ج. "كتفك تحسن 18% ✅ (تحسن حقيقي)" vs "الكاحل +3% (~) (ضمن هامش الخطأ)"
   د. Safety Gates: أي حجب تم رفعه ✅
   هـ. [شارك تقدمك] [حدّث برنامجك]
```

## نقاط مهمة يجب مراعاتها

### 1. الصياغة — Wellness وليس تشخيص

- **لا نقول:** "عندك شد في عضلة كذا" / "عندك مشكلة في المفصل"
- **نقول:** "لاحظنا محدودية في حركة الكتف" / "الحركة تشير لاحتمال شد عضلي"
- **لا نقول:** "عمر جسمك الحركي" (إلا بعد validation كبير)
- **نقول:** "مؤشرات حركتك"
- كل شاشة نتائج فيها: "هذا تقييم حركي — وليس تشخيص طبي"

### 2. الألم = توقف فوري

- زرار "أحس بألم" واضح وسهل الوصول في كل حركة
- عند الضغط: الحركة تتوقف فوراً
- تُسجّل Pain Flag (ليست درجة — بل حالة منفصلة)
- رسالة: "ننصحك باستشارة متخصص قبل التمرين على هذه المنطقة"
- Safety Gate: حجب كامل لتمارين هذه المنطقة

### 3. Visibility Quality Gate

- إذا جودة الرؤية ضعيفة أثناء حركة → الثقة = LOW
- في نهاية الحركة: "الرؤية كانت ضعيفة — هل تريد إعادة الحركة؟"
- إذا رفض → المنطقة = "غير مؤكد" في النتائج (رمادي + "?" في Body Map)

### 4. تحديث Knee-past-toes

الخطة السابقة اعتبرت "تجاوز الركبة لأصابع القدم" خطأ دائماً. هذا ليس دقيق علمياً:
- التجاوز **المحدود** طبيعي ومطلوب في بعض الحركات
- التجاوز **المفرط** (> threshold) مع انهيار الجذع يُعد مؤشر
- النظام يرصد **الدرجة** وليس مجرد "تجاوز ولا لا"

### 5. Movement Quality Grade: 1-3 (والألم منفصل)

| الدرجة | الوصف |
|---|---|
| 3 | حركة نظيفة — لا تعويضات ملحوظة |
| 2 | حركة مع تعويض واحد أو أكثر (لكن الحركة مُكتملة) |
| 1 | فشل — لم يستطع إكمال الحركة أو تعويضات شديدة |
| **Pain** | **حالة منفصلة** — ليست جزء من الـ grade. توقف + Safety Gate |

## أفكار إبداعية — مرتبة حسب الجدوى

### المرحلة القريبة (مع V1)

- **Confidence Badges**: كل مقياس يظهر معه شارة ثقة (HIGH / MEDIUM / LOW)
- **Progress by Real Change**: "تحسن حقيقي ✅" فقط عند تجاوز MDC
- **Hypothesis Cards**: بدل تشخيص مباشر → "ملاحظة + أسباب محتملة"
- **Adaptive Battery**: ابدأ بـ 3، وسّع تلقائياً عند الحاجة

### المرحلة المتوسطة

- **Body Age**: بعد جمع بيانات كافية + validation → "عمر جسمك الحركي" كرقم تحفيزي
- **Weak Spot Auto-Program**: بناء برنامج تصحيحي تلقائي بناءً على النتائج
- **Morning Check-in**: حركة واحدة (30 ثانية) لقياس "جاهزية اليوم"

### المرحلة البعيدة

- **Comparison Insights**: "أفضل من 60% من المتدربين في عمرك" (يحتاج بيانات مجمّعة)
- **Challenge Mode**: كسر الرقم الشخصي في حركة معينة
- **Smartwatch Integration**: HRV + HR + Sleep كمكمّل للتقييم الحركي

## الخلاصة

الـ Body Scan هو الأساس لتخصيص التدريب. البنية التحتية للقياس والتسجيل جاهزة، مما يوفّر بناء محرك القياس من الصفر. لكن الطبقة الجديدة — التفسير والتوصيات — تحتاج بناء دقيق يراعي:

1. **حدود الدقة** — القياس بالكاميرا ليس مثالي، ونحن صريحين بذلك
2. **فرضيات وليس تشخيصات** — كل ملاحظة تُعرض مع أسبابها المحتملة وليس كحقيقة
3. **أمان أولاً** — Safety Gates تمنع أي تصعيد خطير حتى لو المتوسط جيد
4. **تقدم حقيقي** — لا نحتفل إلا بتحسن يتجاوز هامش الخطأ

النظام ليس بديل عن مدرب أو طبيب — لكنه **أفضل بكثير من لا شيء**، وأفضل بكثير من سؤال "ما مستواك؟".
